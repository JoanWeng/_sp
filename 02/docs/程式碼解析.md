# EmoLang 直譯器核心程式碼解析

EmoLang 的直譯器採用了經典的**「前端編譯與樹狀走訪直譯 (Tree-Walk Interpreter)」**設計。最特別的是，它是一套**零相依 (Zero-Dependency)** 的系統級軟體 —— 完全不依賴 C 語言的 `<stdio.h>` 或 `<stdlib.h>`，全靠直接與作業系統 (OS) 溝通來運作！

整個專案拆分為 **3 個標頭檔 (Headers)** 與 **7 個原始碼模組 (Source Files)**。

---

## 📂 1. 專案架構總覽

```
emolang/
├── include/          # 標頭檔目錄
│   ├── emolang.h     # 核心定義 (Token、AST、Value、全域變數)
│   ├── io.h         # I/O 介面宣告
│   └── utils.h       # 工具函式介面與巨集重映射
├── src/              # 原始碼目錄
│   ├── main.c       # 主程式進入點
│   ├── lexer.c      # 詞法分析器 (切詞)
│   ├── parser.c     # 語法分析器 (建 AST)
│   ├── ast.c        # AST 節點記憶體池
│   ├── eval.c       # 求值與執行引擎
│   ├── memory.c     # 記憶體管理與符號表
│   ├── io.c         # OS 系統呼叫包裝
│   └── utils.c      # 自製字串/數學函式
└── emolang           # 編譯好的執行檔
```

## 📂 2. 定義與介面 (include/ 資料夾)

### `emolang.h` (核心藍圖)

整個專案的神經中樞。這裡定義了 EmoLang 的基因：

#### 1️⃣ Token 類型定義 (第 10-20 行)
定義了所有 34 個 Token 類型，分為以下類別：
- **控制流**：`TOK_IF`, `TOK_ELSE`, `TOK_WHILE`, `TOK_FOR`
- **運算符**：`TOK_PLUS`, `TOK_MINUS`, `TOK_MUL`, `TOK_DIV`, `TOK_MOD`
- **比較**：`TOK_EQ`, `TOK_GT`, `TOK_LT`
- **邏輯**：`TOK_AND`, `TOK_OR`, `TOK_NOT`
- **資料結構**：`TOK_LIST`, `TOK_DICT`, `TOK_ARRAY`, `TOK_INDEX`
- **指標與參照**：`TOK_REF`, `TOK_DEREF`, `TOK_DOT`
- **函數相關**：`TOK_FUNC`, `TOK_RETURN`, `TOK_INPUT`
- **基礎類型**：`TOK_NUM`, `TOK_FLOAT_NUM`, `TOK_STR`, `TOK_ID`, `TOK_TRUE`, `TOK_FALSE`

#### 2️⃣ Value 結構 (第 24-25 行) - 動態型別核心
讓 EmoLang 具備類似 Python 的動態型別能力：
```c
typedef struct { int type; int i; double f; char s[256]; } Value;
```
- `type = 0`: 整數/布林
- `type = 1`: 小數 (float)
- `type = 2`: 字串
- `type = 3`: 列表 (使用 list_pool ID)
- `type = 4`: 字典 (使用 dict_pool ID)

#### 3️⃣ AST 節點類型 (第 28-35 行)
對應 Token 的語法樹節點，包含 30+ 種節點類型，涵蓋所有語法結構。

#### 4️⃣ 記憶體與狀態結構 (第 44-48 行)
```c
typedef struct { char name[256]; char fields[20][256]; int field_count; } StructDef;
typedef struct { char name[256]; ASTNode *params; ASTNode *body; } FuncDef;
typedef struct { char name[256]; int addr; } Symbol;
typedef struct { Value items[100]; int count; } ListObj;
typedef struct { char keys[100][256]; Value values[100]; int count; } DictObj;
```

#### 5️⃣ 全域變數宣告 (第 51-58 行)
用來把分散在各個 `.c` 檔的變數牽線綁定在一起：
- `src_code`, `src_pos`: 原始碼位置追蹤
- `memory[MEM_SIZE]`, `heap_ptr`: 虛擬記憶體堆
- `struct_table`, `func_table`: 結構與函數定義表
- `list_pool`, `dict_pool`: 動態資料結構池
- `sym_stack`, `call_depth`: 函數呼叫堆疊 (Scope 實作)
- `ret_val`, `is_returning`: 函數返回值與返回標誌

## 📂 3. 底層引擎 (src/io.c & src/utils.c)

這個區塊是 EmoLang 能脫離 C 標準庫獨立運作的秘密武器。

### `io.c` (作業系統 I/O 通訊)

不使用 `printf` 或 `scanf`，而是直接呼叫 POSIX 作業系統的底層 System Calls：

#### 🔧 系統呼叫宣告 (第 3-9 行)
```c
extern long write(int fd, const void *buf, unsigned long count);
extern long read(int fd, void *buf, unsigned long count);
extern int open(const char *pathname, int flags);
extern long lseek(int fd, long offset, int whence);
extern int close(int fd);
extern void _exit(int status);
```

#### 📤 輸出函式實作 (第 12-29 行)
- **`my_print_str`**: 使用 `write(1, ...)` 向 stdout (fd=1) 寫入
- **`my_print_int`**: 將整數轉為字串後輸出 (手動取餘數)
- **`my_print_float`**: 將浮點數拆分整數與小數部分輸出 (固定 4 位小數)
- **`my_print_newline`**: 輸出換行符 `\n`

#### 📥 輸入函式實作 (第 30-38 行) - `my_read_str`
- 使用 `read(0, ...)` 從 stdin (fd=0) 讀取
- 持續讀取直到遇到 `\n`、`空格`、或達到最大長度
- 自動補上 `\0` 字元結束

#### 🗂️ 檔案操作實作 (第 41-45 行)
```c
int my_open(const char *pathname) { return open(pathname, 0); }  // O_RDONLY
long my_lseek(int fd, long offset, int whence) { return lseek(fd, offset, whence); }
long my_read(int fd, void *buf, unsigned long count) { return read(fd, buf, count); }
void my_close(int fd) { close(fd); }
void my_exit(int status) { _exit(status); }
```

#### 💥 錯誤處理 (第 48-58 行) - `my_error`
統一的崩潰處理器，印出錯誤訊息後發送 `_exit(1)` 中斷程式：
```
❌ 致命錯誤: [訊息] [參數]
```

### `utils.c` (徒手刻出的字串處理器)

完全不使用標準庫，用指標運算刻出所有功能：

#### 字串操作 (第 4-58 行)
- **`my_strlen`**: 逐一計數直到 `\0` (第 4-8 行)
- **`my_strcmp`**: 字元逐一比對，回傳 ASCII 差值 (第 11-14 行)
- **`my_strncmp`**: 限制比對長度的版本 (第 17-24 行)
- **`my_strcpy`**: 經典的 C 極簡寫法 `while ((*d++ = *src++))` (第 27-31 行)
- **`my_strncpy`**: 含補齊 `\0` 的安全版本 (第 34-39 行)
- **`my_strncat`**: 串接字串 (第 42-48 行)
- **`my_strchr`**: 搜尋字元 (第 51-58 行)

#### 字元判斷 (第 62-64 行)
```c
int my_isdigit(char c) { return c >= '0' && c <= '9'; }
int my_isspace(char c) { return c == ' ' || c == '\t' || c == '\n' || c == '\r'; }
```

#### 型別轉換 - 核心演算法 (第 69-120 行)

**`my_atoi`** (第 70-78 行): 字串轉整數
```c
res = res * 10 + (str[i] - '0');  // 利用 ASCII 碼差值
```

**`my_atof`** (第 81-96 行): 字串轉浮點數
- 處理小數點前後
- 小數部分使用 `fraction *= 0.1` 遞減權重
- 支援負號

**`my_itoa`** (第 98-107 行): 整數轉字串
- 取餘數得到每位數字
- 反轉字串陣列

**`my_ftoa`** (第 109-120 行): 浮點數轉字串
- 拆分整數與小數部分
- 分別呼叫 `my_itoa` 和手動輸出小數位

## 📂 4. 前端解析 (src/lexer.c & src/parser.c)

這個區塊負責把人類寫的 Emoji 文字，轉化為電腦能懂的立體結構。

### `lexer.c` (詞法分析器 / 切詞器)

#### 📋 Emoji 關鍵字表 (第 5-20 行)
定義了 41 個 Emoji 與其對應的 Token：
```c
const char *emoji_keywords[] = {
    "📦", "🔢", "🎈", "📝", "🚦", "🟰", "🤔", "🤷", "🔁", "👇", 
    "👆", "📢", "➕", "➖", "✖️", "➗", "✂️", "🤝", "📈", "📉", 
    "🎡", "🚧", "🏗️", "🆕", "➡️", "📍", "🎯", "📚", "📌", "📥", 
    "🟢", "🔴", "🛠️", "🔙", "🔗", "🔀", "🙅",
    "📋", "📖", "🛒", "📏"
};
TokenType emoji_tokens[] = {
    TOK_LET, TOK_LET, TOK_LET, TOK_LET, TOK_LET,
    TOK_ASSIGN, TOK_IF, TOK_ELSE, TOK_WHILE, TOK_LBRACE, TOK_RBRACE, ...
};
```

#### 🔍 `advance_token` 函式 (第 30-81 行)
運作邏輯：逐字元讀取 `src_code`，依序處理：

1. **略過空白** (第 32 行): `while (isspace(src_code[src_pos])) src_pos++;`
2. **EOF 檢查** (第 35 行)
3. **括號與逗號** (第 38-40 行): `(`, `)`, `,`
4. **字串解析** (第 43-51 行): 處理雙引號 `""` 包圍的文字
5. **數字解析** (第 54-62 行): 支援整數與小數 (含 `.` 判斷)
6. **Emoji 關鍵字** (第 65-70 行): 呼叫 `match_keyword` 比對 UTF-8 位元組
7. **普通變數名稱** (第 73-81 行): 識別非 Emoji 的識別符

#### 📌 `eat` 函式 (第 83-85 行)
消費當前 Token 並推進到下一個，若型別不符則呼叫 `my_error` 報錯。

### `parser.c` (語法分析器 / 建樹器)

採用「遞迴下降解析 (Recursive Descent Parsing)」來處理優先級，由低到高分為 9 層：

#### 1️⃣ `parse_primary` (第 7-56 行) - 最高優先級
處理最基礎的語法單元：
- 括號 `()` 包圍的表達式
- 數字 (整數 `TOK_NUM` / 小數 `TOK_FLOAT_NUM`)
- 布林常值 (`TOK_TRUE`, `TOK_FALSE`)
- 字串 `TOK_STR`
- 變數 `TOK_ID`
- **函數呼叫識別**: 如果變數後接 `(`, 判斷為 `AST_FUNC_CALL`

#### 2️⃣ `parse_postfix` (第 59-78 行) - 後綴操作
處理 `.` (結構體成員) 與 `📌` (陣列/字典索引)：
```c
if (current_token.type == TOK_DOT)     → AST_DOT
if (current_token.type == TOK_INDEX)  → AST_INDEX
```

#### 3️⃣ `parse_prefix` (第 81-123 行) - 前綴操作
處理一元運算子：
- `📏` (長度) → `AST_LEN`
- `📥` (輸入) → `AST_INPUT`
- `🙅` (NOT) → `AST_NOT`
- `📍` (取址) → `AST_REF`
- `🎯` (解參照) → `AST_DEREF`
- `🆕` (新建) → `AST_NEW`, `AST_NEW_LIST`, `AST_NEW_DICT`, `AST_ARRAY_ALLOC`

#### 4️⃣ `parse_factor` (第 126-137 行) - 乘除餘
```c
while (TOK_MUL || TOK_DIV || TOK_MOD) → AST_BINOP
```

#### 5️⃣ `parse_addition` (第 140-151 行) - 加減
```c
while (TOK_PLUS || TOK_MINUS) → AST_BINOP
```

#### 6️⃣ `parse_comparison` (第 154-165 行) - 比較
```c
if (TOK_EQ || TOK_GT || TOK_LT) → AST_BINOP
```

#### 7️⃣ `parse_logical_and` (第 168-179 行) - 邏輯 AND
```c
while (TOK_AND) → AST_BINOP
```

#### 8️⃣ `parse_expression` (第 182-193 行) - 邏輯 OR (最終表達式)
```c
while (TOK_OR) → AST_BINOP
```

#### 9️⃣ `parse_block` (第 196-206 行) - 區塊解析
解析 `👇` ... `👆` (即 `{` ... `}`) 包圍的多行語句，回傳語句鏈表。

#### 📜 `parse_statement` (第 209-303 行) - 語句解析
根據開頭 Token 類型分枝處理：

- **`🛠️` (TOK_FUNC)**: 函數定義 → `AST_FUNC_DEF`
  - 解析參數列表 `(param1, param2, ...)`
  - 解析函數主體 `parse_block()`
  
- **`🔙` (TOK_RETURN)**: 返回語句 → `AST_RETURN`

- **`📦` (TOK_LET)**: 變數宣告 → `AST_LET`
  - 可選 `🟰` 賦值

- **`📢` (TOK_PRINT)**: 輸出語句 → `AST_PRINT`

- **`🚦` (TOK_IF)**: 條件判斷 → `AST_IF`
  - 解析條件表達式
  - 解析 true 分支 `parse_block()`
  - 可選 `🤷` (ELSE) 分支

- **`🔁` (TOK_WHILE)**: 迴圈 → `AST_WHILE`
  - 解析條件與迴圈主體

- **`🎡` (TOK_FOR)**: for 迴圈 → `AST_FOR`
  - 解析 init; condition; step 三個部分

- **`🚧` (TOK_STRUCT)**: 結構定義 → `AST_STRUCT_DEF`

- **一般表達式**: 處理賦值 `🟰` 或追加 `🛒`

## 📂 5. 記憶體管理 (src/ast.c & src/memory.c)

### `ast.c` (靜態語法樹池)

#### 📦 靜態記憶體池 (第 3-5 行)
```c
#define MAX_AST_NODES 50000
ASTNode ast_pool[MAX_AST_NODES];
int ast_node_count = 0;
```
因為我們不能使用 `malloc`，所以這裡宣告了一個超大陣列 `ast_pool[50000]`，可容納 50,000 個 AST 節點。

#### 🏭 `create_node` 函式 (第 7-18 行)
每次被呼叫時，就從這個大陣列裡發配一個空的節點出去：
```c
ASTNode *node = &ast_pool[ast_node_count++];
```
這種做法稱為**「靜態記憶體池 (Static Memory Pool)」**，效能極高，無需記憶體配置與釋放。

---

### `memory.c` (虛擬機器記憶體與狀態)

EmoLang 虛擬機的核心儲存區：

#### 🌍 全域變數初始化 (第 6-27 行)
```c
Value memory[MEM_SIZE];        // 10000 格的虛擬記憶體
int heap_ptr = 1;               // 堆記憶體指標

StructDef struct_table[50];    // 結構定義表
FuncDef func_table[50];        // 函數定義表

Symbol sym_stack[20][100];     // 20 層 × 100 符號的呼叫堆疊
int sym_count[20] = {0};
int call_depth = 0;

ListObj list_pool[100];        // 列表物件池 (100 個列表)
DictObj dict_pool[100];       // 字典物件池 (100 個字典)
```

#### 📖 字典操作 (第 34-58 行)
- **`dict_set`**: 設定鍵值，若鍵已存在則覆蓋
- **`dict_get`**: 根據鍵查找值，若不存在則報錯

#### 🗺️ 符號表操作 (第 61-80 行)
- **`alloc_mem`**: 配置記憶體空間，返回位址
- **`get_sym_addr`**: 取得變數位址
  - 先在當前 `call_depth` 層搜尋
  - 若找不到且 `call_depth > 0`，則往全域層 (層 0) 搜尋
  - 若仍找不到，報錯

#### 📍 `get_lvalue` (第 83-103 行) - 取得賦值目標位址
- `AST_VAR`: 從符號表取得位址
- `AST_DEREF`: 對記憶體位址解參照
- `AST_INDEX`: 計算陣列/列表/字典的元素位址
  - 對 List/Dict 使用 `📍` (取址) 會報錯 (它們是動態結構)
- `AST_DOT`: 結構體成員位址計算

## 📂 6. 執行引擎與進入點 (src/eval.c & src/main.c)

### `eval.c` (求值與執行引擎)

這顆大腦負責「走訪 (Traverse)」整棵語法樹並執行動作：

#### 🔮 `is_truthy` (第 4-9 行) - 布林真值判斷
```c
if (v.type == 0) return v.i != 0;        // 整數/布林
if (v.type == 1) return v.f != 0.0;      // 小數
if (v.type == 2) return strlen(v.s) > 0; // 字串
return 0;                                 // 空列表/字典
```

#### 📝 `assign_value` (第 12-56 行) - 統一賦值引擎
支援多種賦值目標：
- `AST_VAR`: 寫入變數
- `AST_DEREF`: 指標解參照寫入
- `AST_DOT`: 結構體成員寫入
- `AST_INDEX`: 
  - `type == 3` (List): 寫入列表元素
  - `type == 4` (Dict): 寫入字典鍵值
  - 其他: 寫入陣列元素

#### 🧮 `eval` (第 59-225 行) - 表達式求值器

核心遞迴函式，依節點類型分枝處理：

1. **NOT**: `AST_NOT` → 取反布林值
2. **基礎類型**:
   - `AST_NUM` → 回傳整數
   - `AST_FLOAT` → 回傳小數
   - `AST_STR` → 回傳字串
   - `AST_TRUE`/`AST_FALSE` → 回傳布林
3. **變數存取**:
   - `AST_VAR` → 從 memory 讀取
   - `AST_REF` → 取位址 (呼叫 `get_lvalue`)
   - `AST_DEREF` → 指標解參照
4. **索引存取** (`AST_INDEX`):
   - `type == 3`: 從 `list_pool` 讀列表元素
   - `type == 4`: 從 `dict_pool` 讀字典值
   - 其他: 從 memory 讀陣列元素
5. **動態建立**:
   - `AST_NEW_LIST` → 建立新列表 ID
   - `AST_NEW_DICT` → 建立新字典 ID
6. **長度運算** (`AST_LEN`): 支援字串、列表、字典
7. **函數呼叫** (`AST_FUNC_CALL`): 第 122-150 行
   - 搜尋 `func_table` 中的函數
   - **遞進 `call_depth`**: `call_depth++`
   - **建立新的符號表層**: `sym_count[call_depth] = 0`
   - **配置參數記憶體**: 逐一 `alloc_mem(1)` 並存參數值
   - **執行函數主體**: `execute(fd->body)`
   - **處理返回值**: 若 `is_returning`，取出 `ret_val`
   - **遞減 `call_depth`**: `call_depth--`
8. **輸入** (`AST_INPUT`): 呼叫 `my_read_str` 讀取並自動判斷型別
9. **記憶體配置**:
   - `AST_ARRAY_ALLOC` → 配置連續陣列
   - `AST_NEW` → 配置結構體 (含欄位空間)
10. **成員存取** (`AST_DOT`): 從 memory 讀取結構成員
11. **雙元運算** (`AST_BINOP`): 第 175-223 行
    - **短路求值**:
      - `TOK_AND`: 左 false → 直接回傳 false
      - `TOK_OR`: 左 true → 直接回傳 true
    - **字串串接**: `TOK_PLUS` 且任一操作數為字串 (type==2)
    - **浮點運算**: 任一操作數為 float (type==1) → 提升為 double 計算
    - **整數運算**: 支援 `+`, `-`, `*`, `/`, `%`, `==`, `>`, `<`

#### 🖨️ `print_value` (第 228-287 行) - 格式化輸出
- `type == 3` (List): 輸出 `[item1, item2, ...]`
- `type == 4` (Dict): 輸出 `{"key": value, ...}`
- `type == 2` (Str): 直接輸出字串
- `type == 1` (Float): 輸出小數
- 其他: 輸出整數/布林

#### ⚙️ `execute` (第 290-347 行) - 程式執行控制器

依序走訪每一行 Statement，呼叫對應處理：

- `AST_RETURN`: 設定 `ret_val`，設 `is_returning = 1` 中斷
- `AST_FUNC_DEF`: 註冊到 `func_table`
- `AST_LET`: 配置記憶體並初始化 (含宣告變數)
- `AST_ASSIGN`: 呼叫 `assign_value` 寫入
- `AST_APPEND`: 將元素推入列表 (`🛒`)
- `AST_STRUCT_DEF`: 解析結構欄位並註冊
- `AST_PRINT`: 呼叫 `print_value` 輸出
- `AST_IF`: 條件分支執行
- `AST_WHILE`: 迴圈執行
- `AST_FOR`: for 迴圈執行 (init → cond → body → step)

## 📌 特色與亮點總結

| 特性 | 說明 |
|------|------|
| **零相依** | 不使用 `stdio.h`, `stdlib.h`，全用 POSIX 系統呼叫 |
| **靜態記憶體池** | AST 節點使用 `ast_pool[50000]`，高效無碎片 |
| **虛擬記憶體** | `memory[10000]` 模擬 Heap，透過 `heap_ptr` 配置 |
| **Scope 隔離** | `sym_stack[20][100]` + `call_depth` 實現函數區域變數 |
| **短路求值** | `&&` 與 `||` 遇 false/true 直接回傳 |
| **動態型別** | Value 結構支援 5 種型別 (int/float/str/list/dict) |
| **巨集重映射** | `#define strlen my_strlen` 自動轉向自訂函式 |
| **UTF-8 Emoji** | 支援 41 個 Emoji 關鍵字作為語法 |
