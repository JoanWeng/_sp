# 第 1 章　系統程式概論

> 本章目標：了解系統程式的定義、分類、發展歷史與程式從原始碼到執行的完整流程。

---

## 1.1 什麼是系統程式？

**系統程式（System Programs / System Software）** 是介於電腦硬體與應用程式之間的一層軟體。它的主要任務是：

- 管理電腦的硬體資源（CPU、記憶體、I/O 裝置）
- 提供應用程式開發所需的工具與環境
- 作為使用者與硬體之間的橋樑

### 系統程式 vs 應用程式

| 比較項目 | 系統程式 | 應用程式 |
|----------|----------|----------|
| 目的 | 管理資源、提供平台 | 解決特定使用者問題 |
| 使用者 | 開發者、作業系統 | 一般終端使用者 |
| 範例 | 編譯器、作業系統、連結器 | Word、瀏覽器、遊戲 |
| 與硬體的距離 | 近 | 遠 |

---

## 1.2 系統程式的分類

### 主要類型

```
系統程式
├── 語言處理程式
│   ├── 組譯器 (Assembler)
│   ├── 編譯器 (Compiler)
│   ├── 直譯器 (Interpreter)
│   └── 巨集處理器 (Macro Processor)
├── 程式開發工具
│   ├── 連結器 (Linker)
│   ├── 載入器 (Loader)
│   ├── 偵錯器 (Debugger)
│   └── 程式庫 (Library)
└── 作業系統 (Operating System)
    ├── 行程管理
    ├── 記憶體管理
    ├── 檔案系統
    └── 裝置驅動程式
```

### 各類別說明

| 類型 | 功能 | 常見工具 |
|------|------|----------|
| **組譯器** | 將組合語言翻譯為機器碼 | NASM、MASM、GAS |
| **編譯器** | 將高階語言翻譯為目的碼 | GCC、Clang、MSVC |
| **直譯器** | 逐行解讀並執行原始碼 | Python、Ruby、Shell |
| **巨集處理器** | 展開巨集定義、文字替換 | m4、C Preprocessor |
| **連結器** | 合併目的碼、解析符號 | ld、link.exe |
| **載入器** | 將程式載入記憶體執行 | OS 內建 |
| **偵錯器** | 協助程式設計師除錯 | GDB、LLDB |
| **作業系統** | 管理所有硬體與軟體資源 | Linux、Windows、macOS |

## 1.3 電腦系統的層次架構

```
┌─────────────────────────────────┐
│         應用程式層               │  ← Word、Chrome、遊戲
├─────────────────────────────────┤
│       程式語言 / 程式庫層         │  ← C 標準庫、Java SDK
├─────────────────────────────────┤
│         作業系統層               │  ← Linux、Windows
├─────────────────────────────────┤
│       系統程式工具層              │  ← 編譯器、連結器、組譯器
├─────────────────────────────────┤
│       機器語言 / 韌體層           │  ← BIOS、UEFI
├─────────────────────────────────┤
│           硬體層                 │  ← CPU、RAM、磁碟
└─────────────────────────────────┘
```

## 1.4 程式的執行流程

從一個 `.c` 原始碼到最終在記憶體中執行，需要經過以下步驟：

### 完整流程

```
原始碼 hello.c
    │
    ▼ [前置處理器 Preprocessor]
    │  展開 #include、#define、條件編譯
    │
展開後程式碼 hello.i
    │
    ▼ [編譯器 Compiler]
    │  語法分析 → 語意分析 → 最佳化 → 產生組語
    │
組合語言 hello.s
    │
    ▼ [組譯器 Assembler]
    │  將組語翻譯為機器碼，產生可重定位目的碼
    │
目的碼 hello.o
    │
    ▼ [連結器 Linker]
    │  合併多個 .o 檔、解析符號、加入程式庫
    │
可執行檔 hello (ELF / PE)
    │
    ▼ [載入器 Loader]
    │  配置記憶體、載入程式、設定執行環境
    │
記憶體中執行 (Process)
```

### 用 GCC 觀察各階段

```bash
gcc -E hello.c -o hello.i      # 只做前置處理
gcc -S hello.c -o hello.s      # 產生組合語言
gcc -c hello.c -o hello.o      # 產生目的碼
gcc    hello.c -o hello         # 完整編譯與連結
```

## 1.5 機器語言與組合語言

### 機器語言（Machine Language）
- CPU 能直接執行的二進位指令
- 完全依賴硬體架構，不具可攜性
- 範例（x86）：`B8 05 00 00 00` = 將數值 5 載入 EAX 暫存器

### 組合語言（Assembly Language）
- 機器語言的符號化表示（助記符）
- 一條組語指令對應一條（或少數幾條）機器指令
- 範例：`MOV EAX, 5`

### 高階語言（High-level Language）
- 接近人類思維，與硬體無關
- 需編譯器翻譯為機器碼
- 範例：`int x = 5;`

| 層次 | 範例 | 可讀性 | 效能控制 |
|------|------|--------|----------|
| 高階語言 | `x = a + b` | 高 | 低 |
| 組合語言 | `MOV EAX, [a]` | 中 | 高 |
| 機器語言 | `8B 05 ...` | 低 | 最高 |

## 1.6 系統程式的發展歷史

| 時期 | 重要發展 |
|------|----------|
| **1940s** | 電腦使用機器語言，直接輸入二進位碼 |
| **1950s** | 組合語言出現，第一個組譯器誕生（IBM 701） |
| **1950s末** | FORTRAN 問世，第一個高階語言編譯器（1957） |
| **1960s** | 作業系統概念形成（IBM OS/360），連結器標準化 |
| **1970s** | Unix 誕生（1969-1970），C 語言發明，系統程式以高階語言撰寫 |
| **1980s** | PC 普及，DOS/Windows，GNU 工具鏈（GCC、GDB） |
| **1990s** | Linux 核心（1991），開放原始碼蓬勃發展 |
| **2000s後** | LLVM/Clang 架構，JIT 編譯技術，容器與虛擬化 |

## 1.7 重要術語整理

| 術語 | 英文 | 定義 |
|------|------|------|
| 原始碼 | Source Code | 人類撰寫的程式文字 |
| 目的碼 | Object Code | 組譯/編譯後的可重定位機器碼 |
| 可執行檔 | Executable | 可直接執行的完整程式 |
| 符號 | Symbol | 函式或變數的名稱 |
| 符號表 | Symbol Table | 記錄所有符號與其位址的表格 |
| 重定位 | Relocation | 調整程式碼中的位址以符合載入位置 |
| 位置計數器 | Location Counter | 追蹤當前組譯位址的計數器 |
| 程式庫 | Library | 可被多個程式共享的目的碼集合 |
| ABI | Application Binary Interface | 定義程式在二進位層面互動的規範 |
| ISA | Instruction Set Architecture | 定義 CPU 支援的指令集 |

---

## 1.8 本章小結

- 系統程式是電腦系統的基礎建設，介於硬體與應用程式之間
- 主要包含：組譯器、編譯器、直譯器、巨集處理器、連結器、載入器、作業系統
- 程式的執行需要經過：前置處理 → 編譯 → 組譯 → 連結 → 載入 → 執行
- 了解系統程式有助於撰寫高效、正確的程式，並理解底層運作機制
