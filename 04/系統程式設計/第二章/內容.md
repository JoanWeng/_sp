# 第 2 章　組合語言基礎

> 本章目標：掌握組合語言的程式結構、資料定義方式、虛擬指令，以及基本的 x86 組語程式框架。

---

## 2.1 組合語言的基本概念

### 為什麼學組合語言？

1. 理解 CPU 的實際運作方式
2. 撰寫效能極致的程式碼（嵌入式、驅動程式）
3. 逆向工程與資安分析的基礎
4. 深入理解編譯器輸出結果

### 組語程式的組成要素

每一行組合語言程式碼由最多四個欄位組成：

```
[標號:]  助記符  [運算元]  [; 註解]
```

| 欄位 | 說明 | 是否必要 |
|------|------|----------|
| **標號 (Label)** | 代表該行位址的名稱，以 `:` 結尾 | 選擇性 |
| **助記符 (Mnemonic)** | 指令名稱（如 MOV、ADD）或虛擬指令 | 必要 |
| **運算元 (Operand)** | 指令的操作對象，可有 0~2 個 | 視指令而定 |
| **註解 (Comment)** | `;` 之後到行尾，組譯器忽略 | 選擇性 |

### 範例說明

```asm
START:  MOV  EAX, 10    ; 將 10 載入 EAX 暫存器
        ADD  EAX, EBX   ; EAX = EAX + EBX
        JMP  START      ; 跳回 START 標號
```

## 2.2 程式區段（Section）

組合語言程式通常分為三個區段：

### NASM 格式

```asm
section .data           ; 已初始化資料區段
    msg  db "Hello!", 0x0a
    num  dd 42

section .bss            ; 未初始化資料區段（執行時配置）
    buf  resb 256       ; 保留 256 bytes

section .text           ; 程式碼區段
    global _start       ; 宣告進入點為全域符號

_start:
    ; 程式邏輯從這裡開始
```

### MASM / Windows 格式

```asm
.data
    msg  BYTE "Hello!", 0
    num  DWORD 42

.code
main PROC
    ; 程式碼
    ret
main ENDP
END main
```

### 三個區段的特性比較

| 區段 | 名稱 | 內容 | 執行時是否佔用記憶體 |
|------|------|------|---------------------|
| `.data` | 資料區段 | 已初始化變數 | 是（寫入磁碟） |
| `.bss` | BSS 區段 | 未初始化變數 | 是（執行時清零） |
| `.text` | 程式碼區段 | 可執行指令 | 是（唯讀） |

> **BSS** 全名：Block Started by Symbol，程式啟動時由作業系統自動清為 0。

## 2.3 資料定義虛擬指令

虛擬指令（Assembler Directives / Pseudo-instructions）不是真正的 CPU 指令，而是給組譯器的指示。

### 資料定義指令（NASM 風格）

| 指令 | 大小 | 範例 | 說明 |
|------|------|------|------|
| `DB` | 1 byte | `x DB 10` | 定義 Byte |
| `DW` | 2 bytes | `y DW 1000` | 定義 Word |
| `DD` | 4 bytes | `z DD 0` | 定義 Double Word |
| `DQ` | 8 bytes | `q DQ 0` | 定義 Quad Word |
| `DT` | 10 bytes | `t DT 0` | 定義 Ten Bytes |

### 空間保留指令（未初始化）

| 指令 | 說明 | 範例 |
|------|------|------|
| `RESB n` | 保留 n bytes | `buf RESB 64` |
| `RESW n` | 保留 n words | `buf RESW 32` |
| `RESD n` | 保留 n dwords | `buf RESD 16` |
| `RESQ n` | 保留 n qwords | `buf RESQ 8` |

### 常數定義

```asm
MAX     EQU  100         ; 常數，不佔記憶體空間
NEWLINE EQU  0x0a        ; 換行字元
SIZE    EQU  $ - buffer  ; $ 代表目前位址，計算長度
```

### 資料定義範例

```asm
section .data
    ; 數值
    byte_val    DB  255          ; 單一 byte
    word_val    DW  1000         ; 2 bytes
    dword_val   DD  100000       ; 4 bytes

    ; 字串（字元陣列）
    msg         DB  "Hello", 0   ; C 風格字串（以 0 結尾）
    msg_len     EQU $ - msg      ; 自動計算長度

    ; 陣列
    arr         DD  1, 2, 3, 4, 5   ; 5 個 DWORD 的陣列
    arr_len     EQU ($ - arr) / 4   ; 元素個數

    ; 十六進位、二進位
    hex_val     DB  0xFF         ; 255（十六進位）
    bin_val     DB  0b10101010   ; 170（二進位）

section .bss
    input_buf   RESB 128         ; 128 bytes 輸入緩衝區
    result      RESD 1           ; 1 個 DWORD 的空間
```

## 2.4 常用虛擬指令總覽

### 程式控制類

| 指令 | 說明 | 範例 |
|------|------|------|
| `GLOBAL` | 宣告為全域符號（可被外部引用） | `global _start` |
| `EXTERN` | 宣告外部符號（定義在其他檔案） | `extern printf` |
| `ORG` | 設定程式起始位址 | `ORG 0x100` |
| `BITS` | 設定產生 16/32/64 位元碼 | `BITS 64` |
| `SECTION` | 切換到指定區段 | `section .text` |

### 巨集與條件類

```asm
%define MAX 100             ; 文字替換巨集
%ifdef DEBUG                ; 條件組譯（若 DEBUG 已定義）
    ; 除錯程式碼
%endif

%include "macros.asm"       ; 引入其他組語檔案
```

## 2.5 數值表示方式

組譯器支援多種數值格式：

```asm
; 十進位（預設）
mov eax, 255

; 十六進位
mov eax, 0xFF       ; NASM 風格
mov eax, 0FFh       ; MASM 風格（需以數字開頭）

; 二進位
mov eax, 0b11111111  ; NASM 風格
mov eax, 11111111b   ; MASM 風格

; 八進位
mov eax, 0o377       ; NASM 風格
mov eax, 377o        ; 八進位 377 = 十進位 255

; 字元（ASCII 值）
mov al, 'A'          ; al = 65
mov ax, 'AB'         ; ax = 0x4241（小端序）
```

## 2.6 標號（Label）的使用

```asm
; 全域標號（函式進入點）
_start:
    mov ecx, 5          ; 計數器設為 5

; 迴圈標號
loop_begin:
    dec ecx
    jnz loop_begin      ; 若 ECX ≠ 0 則跳回

; 本地標號（NASM 以 . 開頭，避免命名衝突）
.local_label:
    nop

; 前向參考（使用尚未定義的標號）
    jmp end_of_program  ; end_of_program 在後面才定義

; ... 中間程式碼 ...

end_of_program:
    ; 組譯器在第二遍解析時才能確定位址
```

## 2.7 完整程式範例：Hello World

### Linux x86-64（NASM）

```asm
; hello.asm - 在 Linux 上印出 "Hello, World!"
; 組譯：nasm -f elf64 hello.asm -o hello.o
; 連結：ld hello.o -o hello
; 執行：./hello

BITS 64
section .data
    msg     DB  "Hello, World!", 0x0a   ; 字串 + 換行
    msg_len EQU $ - msg                  ; 計算字串長度（14）

section .text
    global _start

_start:
    ; 系統呼叫：write(1, msg, msg_len)
    mov rax, 1          ; sys_write 編號
    mov rdi, 1          ; fd = 1（stdout）
    mov rsi, msg        ; 字串位址
    mov rdx, msg_len    ; 字串長度
    syscall             ; 呼叫核心

    ; 系統呼叫：exit(0)
    mov rax, 60         ; sys_exit 編號
    xor rdi, rdi        ; 結束碼 = 0
    syscall
```

### 程式解析

| 指令 | 說明 |
|------|------|
| `BITS 64` | 產生 64 位元機器碼 |
| `DB "Hello, World!", 0x0a` | 定義字串資料，`0x0a` 為換行 |
| `EQU $ - msg` | `$` 為目前位址，相減得字串長度 |
| `global _start` | 告訴連結器程式進入點 |
| `mov rax, 1` | 設定系統呼叫編號（write） |
| `syscall` | 觸發 64 位元 Linux 系統呼叫 |
| `xor rdi, rdi` | 比 `mov rdi, 0` 更短的清零方式 |

## 2.8 組譯過程：兩遍掃描

組譯器通常需要**兩次掃描**原始碼才能完成組譯：

### 第一遍（Pass 1）：建立符號表

```
目的：
1. 計算每個標號的位址
2. 建立符號表（Symbol Table）
3. 確定各指令的長度

位置計數器（Location Counter, LC）：
- 初始值通常為 0 或 ORG 指定的值
- 每處理一條指令後，LC += 該指令的位元組數
```

### 第二遍（Pass 2）：產生目的碼

```
目的：
1. 利用符號表，翻譯所有指令為機器碼
2. 解析前向參考（Forward Reference）
3. 輸出目的碼與重定位資訊
```

### 兩遍組譯範例

```asm
位址  原始碼              第一遍：LC 計算
0000  START: MOV EAX,5   LC=0,  記錄 START=0,  LC+=5 → 0005
0005  ADD EAX, EBX       LC=5,                  LC+=2 → 0007
0007  JMP END            LC=7,  END 未知（前向參考）,  LC+=5 → 000C
000C  END: NOP            LC=12, 記錄 END=12,    LC+=1 → 000D

第二遍：
JMP END → JMP 0x0C  (現在知道 END 的位址了)
```

## 2.9 本章小結

- 組語程式由**標號、助記符、運算元、註解**四個欄位組成
- 程式分為三大區段：`.data`（已初始化資料）、`.bss`（未初始化資料）、`.text`（程式碼）
- 虛擬指令（如 `DB`、`EQU`、`RESB`）是給組譯器的指示，不產生執行指令
- 組譯器採用**兩遍掃描**：第一遍建立符號表，第二遍產生目的碼
- `$` 符號代表目前組譯位址，常用於計算資料長度
