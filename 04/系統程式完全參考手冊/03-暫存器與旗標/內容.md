# 第 3 章　暫存器與旗標

> **本章目標：** 掌握 x86-64 架構中所有暫存器的分類、命名規則、用途與存取方式，理解旗標暫存器（FLAGS）每個位元的意義，以及旗標如何影響條件跳躍指令的行為。
>
> **預備知識：** 第 2 章組合語言基礎、二補數、基本位元運算。

## 3.1 什麼是暫存器？

### 3.1.1 暫存器的定位

CPU 在執行運算時，資料必須先從記憶體載入到 CPU 內部才能處理。而 CPU 內部儲存資料的地方就是**暫存器（Register）**。

```
資料移動速度（由快到慢）：

暫存器（Register）    ← 在 CPU 晶片內部，存取時間 < 1 個時脈週期
     ↕
L1 快取（L1 Cache）   ← 在 CPU 晶片內，~4 個時脈週期
     ↕
L2 / L3 快取         ← 在 CPU 晶片內或附近，~10～40 個時脈週期
     ↕
主記憶體（RAM）       ← 在主機板上，~100～200 個時脈週期
     ↕
硬碟 / SSD           ← 外部儲存，~100,000～幾百萬個時脈週期
```

暫存器是 CPU 能直接操作的**最快儲存單元**。x86-64 的大多數指令只能對暫存器或記憶體操作，而不能直接對兩個記憶體位址同時操作：

```asm
ADD EAX, EBX        ; ✓ 合法：兩個都是暫存器
ADD EAX, [var]      ; ✓ 合法：暫存器 + 記憶體
ADD [var1], [var2]  ; ✗ 不合法：不能兩個都是記憶體位址
```

### 3.1.2 x86-64 暫存器概覽

x86-64 架構的暫存器可分為以下幾大類：

```
x86-64 暫存器
├── 通用暫存器（General-Purpose Registers, GPR）  × 16 個，各 64-bit
├── 旗標暫存器（RFLAGS）                           × 1 個，64-bit
├── 指令指標暫存器（RIP）                           × 1 個，64-bit
├── 區段暫存器（Segment Registers）                × 6 個，各 16-bit
├── x87 FPU 暫存器（ST0～ST7）                     × 8 個，各 80-bit
├── MMX 暫存器（MM0～MM7）                         × 8 個，各 64-bit（與 ST 共用）
└── SSE/AVX 暫存器（XMM/YMM/ZMM）                 × 16～32 個
```

本章重點介紹最常使用的**通用暫存器**與**旗標暫存器**。

## 3.2 通用暫存器（General-Purpose Registers）

### 3.2.1 64-bit 時代的暫存器擴展歷史

x86 架構的暫存器經歷了三次主要擴展：

```
8086（1978）    → 16-bit 暫存器：AX, BX, CX, DX, SP, BP, SI, DI
     ↓ 擴展
i386（1985）    → 32-bit 暫存器：EAX, EBX, ECX, EDX, ESP, EBP, ESI, EDI（在前面加 E）
     ↓ 擴展
x86-64（2003）  → 64-bit 暫存器：RAX, RBX, RCX, RDX, RSP, RBP, RSI, RDI（在前面加 R）
                  + 新增 8 個：R8, R9, R10, R11, R12, R13, R14, R15
```

### 3.2.2 暫存器的子部分存取

x86-64 的通用暫存器保留了向下相容性，可以用不同名稱存取同一個暫存器的不同部分：

```
RAX（64-bit）
┌───────────────────────────────────────────────────────────────┐
│ 63                                                           0 │
│                              RAX                              │
│                         ┌─────────────────────────────────── │
│                         │              EAX（32-bit）          │
│                         │         ┌───────────────────────── │
│                         │         │         AX（16-bit）      │
│                         │         │    ┌────────┬─────────── │
│                         │         │    │   AH   │     AL     │
│                         │         │    │ 15～8  │    7～0    │
└─────────────────────────┴─────────┴────┴────────┴────────────┘
```

以 RAX 為例，四個名稱代表同一個實體暫存器的不同位元範圍：

| 名稱 | 位元範圍 | 大小 | 說明 |
|------|---------|------|------|
| `RAX` | 63 ～ 0 | 64-bit | 完整暫存器（x86-64 新增） |
| `EAX` | 31 ～ 0 | 32-bit | 低 32 位元（i386 引入） |
| `AX` | 15 ～ 0 | 16-bit | 低 16 位元（8086 原始） |
| `AH` | 15 ～ 8 | 8-bit | AX 的高 8 位元 |
| `AL` | 7 ～ 0 | 8-bit | AX 的低 8 位元 |

**重要：寫入 EAX 會清除 RAX 的高 32 位元！**

```asm
mov rax, 0xFFFFFFFFFFFFFFFF  ; RAX = 全 1
mov eax, 0                   ; 寫入 EAX = 0
; 結果：RAX = 0x0000000000000000  （高 32 位元被清除）
; 這是 x86-64 的設計：寫入 32-bit 暫存器會 zero-extend 到 64-bit

mov rax, 0xFFFFFFFFFFFFFFFF  ; RAX = 全 1
mov ax, 0                    ; 只寫入低 16 位元
; 結果：RAX = 0xFFFFFFFFFFFF0000  （高 48 位元保留）

mov rax, 0xFFFFFFFFFFFFFFFF  ; RAX = 全 1
mov al, 0                    ; 只寫入最低 8 位元
; 結果：RAX = 0xFFFFFFFFFFFFFF00  （高 56 位元保留）
```

### 3.2.3 十六個通用暫存器完整說明

#### 原始八個暫存器（8086 遺留，各有傳統用途）

| 64-bit | 32-bit | 16-bit | 8-bit 低 | 8-bit 高 | 傳統用途 | 常見現代用途 |
|--------|--------|--------|---------|---------|---------|------------|
| `RAX` | `EAX` | `AX` | `AL` | `AH` | 累加器（Accumulator） | 函式回傳值、系統呼叫編號 |
| `RBX` | `EBX` | `BX` | `BL` | `BH` | 基底暫存器（Base） | 通用、callee-saved |
| `RCX` | `ECX` | `CX` | `CL` | `CH` | 計數暫存器（Counter） | 迴圈計數、移位次數、第 4 參數 |
| `RDX` | `EDX` | `DX` | `DL` | `DH` | 資料暫存器（Data） | 乘除法擴展、第 3 參數 |
| `RSP` | `ESP` | `SP` | `SPL` | — | 堆疊指標（Stack Pointer） | **始終指向堆疊頂端** |
| `RBP` | `EBP` | `BP` | `BPL` | — | 基底指標（Base Pointer） | 堆疊框架基底、callee-saved |
| `RSI` | `ESI` | `SI` | `SIL` | — | 來源索引（Source Index） | 字串操作來源、第 2 參數 |
| `RDI` | `EDI` | `DI` | `DIL` | — | 目的索引（Destination Index） | 字串操作目的、第 1 參數 |

#### x86-64 新增八個暫存器

| 64-bit | 32-bit | 16-bit | 8-bit | 用途 |
|--------|--------|--------|-------|------|
| `R8` | `R8D` | `R8W` | `R8B` | 第 5 個函式參數 |
| `R9` | `R9D` | `R9W` | `R9B` | 第 6 個函式參數 |
| `R10` | `R10D` | `R10W` | `R10B` | caller-saved 通用暫存器 |
| `R11` | `R11D` | `R11W` | `R11B` | caller-saved 通用暫存器 |
| `R12` | `R12D` | `R12W` | `R12B` | callee-saved 通用暫存器 |
| `R13` | `R13D` | `R13W` | `R13B` | callee-saved 通用暫存器 |
| `R14` | `R14D` | `R14W` | `R14B` | callee-saved 通用暫存器 |
| `R15` | `R15D` | `R15W` | `R15B` | callee-saved 通用暫存器 |

> **R8D、R8W、R8B 的命名規則：**
> 新增暫存器以數字命名，後綴 D（Dword）、W（Word）、B（Byte）表示寬度。

### 3.2.4 各暫存器詳細說明

#### RAX — 累加器（Accumulator）

```asm
; 1. 函式回傳值（Linux x86-64 ABI）
call my_function
; 呼叫後 RAX 存放回傳值

; 2. 系統呼叫編號
mov rax, 1          ; sys_write
syscall

; 3. 乘法的隱含目的暫存器
mov eax, 100
mov ebx, 200
mul ebx             ; 無號乘法：EDX:EAX = EAX × EBX
                    ; 結果 = 20000，存入 EAX（高位存 EDX）

; 4. 除法的隱含來源
mov eax, 1000
mov edx, 0          ; EDX:EAX = 1000
mov ecx, 7
div ecx             ; EAX = 商 = 142，EDX = 餘數 = 6
```

#### RCX — 計數暫存器（Counter）

```asm
; 1. LOOP 指令隱含使用 ECX
mov ecx, 5          ; 迴圈 5 次
my_loop:
    ; ... 迴圈體 ...
    loop my_loop    ; ECX--；若 ECX ≠ 0 則跳回 my_loop

; 2. 移位/旋轉指令的位移量
mov cl, 3           ; 移位 3 位
shl eax, cl         ; EAX 左移 CL 位（使用 CL 而非整個 CX）

; 3. REP 前綴指令（字串操作）
mov ecx, 100        ; 重複 100 次
rep movsb           ; 重複執行 MOVSB（複製 100 個 bytes）
```

#### RSP — 堆疊指標（Stack Pointer）

```asm
; RSP 始終指向堆疊頂端（最後一個 PUSH 的資料）
; 堆疊向低位址成長！

push rax            ; RSP -= 8，然後 [RSP] = RAX
pop  rbx            ; RBX = [RSP]，然後 RSP += 8

; ⚠️ 永遠不要用 RSP 儲存其他資料，
;    函式呼叫與中斷機制完全依賴 RSP 的正確性
```

#### RBP — 基底指標（Base Pointer）

```asm
; RBP 通常用來建立「堆疊框架（Stack Frame）」
; 讓函式能固定位址存取區域變數與參數

my_function:
    push rbp            ; 保存呼叫者的 RBP
    mov  rbp, rsp       ; 設定本函式的堆疊框架基底
    sub  rsp, 16        ; 保留 16 bytes 給區域變數

    ; 現在可以用 [rbp-4]、[rbp-8] 等存取區域變數
    mov dword [rbp-4],  10  ; 第一個區域變數 = 10
    mov dword [rbp-8],  20  ; 第二個區域變數 = 20

    mov rsp, rbp        ; 釋放區域變數空間
    pop rbp             ; 還原呼叫者的 RBP
    ret
```

#### RSI / RDI — 索引暫存器

```asm
; 字串/記憶體操作的來源與目的
; RSI = Source Index（來源）
; RDI = Destination Index（目的）

mov rsi, src_addr   ; 來源位址
mov rdi, dst_addr   ; 目的位址
mov rcx, 100        ; 複製 100 bytes
rep movsb           ; 每次從 [RSI] 複製到 [RDI]，RSI++，RDI++

; Linux x86-64 ABI：第一、二個函式參數
; func(arg1, arg2) → arg1 在 RDI，arg2 在 RSI
```

## 3.3 Linux x86-64 函式呼叫慣例（ABI）

暫存器的使用並非完全自由，函式之間有一套共同遵守的規範。

### 3.3.1 參數傳遞順序

Linux x86-64 使用 **System V AMD64 ABI**，整數/指標參數依序存入：

```
第 1 個參數 → RDI
第 2 個參數 → RSI
第 3 個參數 → RDX
第 4 個參數 → RCX
第 5 個參數 → R8
第 6 個參數 → R9
第 7 個以上 → 透過堆疊傳遞（由右至左壓入）

回傳值      → RAX（64-bit 以內）
             RAX + RDX（128-bit）
```

**對應 C 程式碼：**

```c
// C 宣告
long add(long a, long b, long c);
// 呼叫 add(10, 20, 30)
```

```asm
; 組語對應
mov rdi, 10         ; 第 1 個參數 a = 10
mov rsi, 20         ; 第 2 個參數 b = 20
mov rdx, 30         ; 第 3 個參數 c = 30
call add
; 執行後 RAX = 回傳值
```

### 3.3.2 Caller-saved vs Callee-saved

```
Caller-saved（呼叫者負責保存，被呼叫者可以隨意修改）：
  RAX, RCX, RDX, RSI, RDI, R8, R9, R10, R11

Callee-saved（被呼叫者負責保存，使用前必須 PUSH，返回前 POP）：
  RBX, RBP, R12, R13, R14, R15
  RSP（始終保持呼叫時的對齊狀態）
```

```asm
my_func:
    ; 若要使用 RBX，必須先保存
    push rbx
    push r12

    mov rbx, 100    ; 現在可以使用 RBX
    mov r12, 200    ; 現在可以使用 R12

    ; ... 函式主體 ...

    ; 返回前還原（順序相反！）
    pop r12
    pop rbx
    ret
```

## 3.4 特殊用途暫存器

### 3.4.1 RIP — 指令指標（Instruction Pointer）

RIP（32-bit: EIP，16-bit: IP）存放**下一條要執行指令的位址**，也稱為程式計數器（Program Counter, PC）。

```
CPU 執行週期：
  1. 從 [RIP] 讀取指令
  2. RIP += 指令長度（指向下一條指令）
  3. 解碼並執行指令
  4. 重複

程式設計師通常不能直接修改 RIP，
只能透過跳躍/呼叫指令間接改變它：

JMP  label    → RIP = label 的位址
CALL func     → PUSH RIP；RIP = func 的位址
RET           → POP RIP
```

**RIP 相對定址（x86-64 重要特性）：**

```asm
; x86-64 新增了 RIP 相對定址模式
; 允許以 RIP 為基準存取資料，方便 PIC（位置無關程式碼）

mov rax, [rip + my_var]   ; my_var 的位址 = RIP + 偏移量
; 好處：無論程式載入到哪個位址，偏移量是固定的
```

### 3.4.2 區段暫存器（Segment Registers）

在 64-bit 模式下，區段暫存器已大幅簡化，但仍存在：

| 暫存器 | 全名 | 現代用途 |
|--------|------|---------|
| `CS` | Code Segment | 程式碼區段（由 OS 管理，不直接使用） |
| `DS` | Data Segment | 資料區段（通常不需要設定） |
| `ES` | Extra Segment | 字串操作目的（部分指令仍使用）|
| `FS` | — | **Thread-Local Storage（TLS）** 指標 |
| `GS` | — | OS 核心內部使用（如 Linux 的 percpu 資料）|
| `SS` | Stack Segment | 堆疊區段（由 OS 管理） |

> 在 64-bit 保護模式下，大多數的記憶體保護由分頁（Paging）機制負責，區段暫存器的重要性已大幅降低。`FS` 和 `GS` 是例外，用於執行緒本地儲存。

## 3.5 旗標暫存器（RFLAGS）

### 3.5.1 旗標暫存器的結構

RFLAGS 是 64-bit 暫存器，其中每個位元代表一個旗標（Flag）。大多數指令執行後會**自動更新**部分旗標，反映運算結果的狀態。

```
RFLAGS 位元佈局（僅列出常用位元）：

位元 63 ～ 22：保留（AMD64 未使用）
位元 21：ID    — CPUID 指令可用旗標
位元 18：AC    — Alignment Check
位元 17：VM    — Virtual-8086 Mode
位元 16：RF    — Resume Flag
位元 14：NT    — Nested Task
位元 13～12：IOPL — I/O Privilege Level
位元 11：OF   ← 溢位旗標（Overflow Flag）
位元 10：DF   ← 方向旗標（Direction Flag）
位元  9：IF   ← 中斷致能旗標（Interrupt Enable Flag）
位元  8：TF    — Trap Flag（單步除錯）
位元  7：SF   ← 符號旗標（Sign Flag）
位元  6：ZF   ← 零旗標（Zero Flag）
位元  4：AF   ← 輔助進位旗標（Auxiliary Carry Flag）
位元  2：PF   ← 奇偶旗標（Parity Flag）
位元  0：CF   ← 進位旗標（Carry Flag）
```

### 3.5.2 六個核心旗標詳解

#### CF — Carry Flag（進位旗標）

記錄**無號數運算**是否發生進位或借位。

```asm
; 加法進位
mov al, 0xFF    ; AL = 255
add al, 1       ; AL = 0（溢出），CF = 1（發生進位）

mov al, 0x10    ; AL = 16
add al, 0x05    ; AL = 21，CF = 0（未進位）

; 減法借位
mov al, 5
sub al, 10      ; AL = 251（有借位），CF = 1

; 移位操作
mov al, 0b10110001
shr al, 1       ; 右移 1 位，最低位 (1) 移入 CF，CF = 1
                ; AL = 0b01011000
```

**CF 的用途：**
- 多精度加法（ADC = Add with Carry）
- 多精度減法（SBB = Subtract with Borrow）
- 判斷無號數大小（JC / JNC / JB / JAE）

#### ZF — Zero Flag（零旗標）

運算結果為**零**時置 1，否則為 0。

```asm
mov eax, 5
sub eax, 5      ; EAX = 0，ZF = 1

mov eax, 5
sub eax, 3      ; EAX = 2，ZF = 0

; CMP 指令本質是 SUB，但不存結果，只設旗標
cmp eax, eax    ; ZF = 1（任何值減自己都為 0）
cmp eax, 0      ; 若 EAX = 0 則 ZF = 1

; AND 操作後的 ZF
test eax, eax   ; EAX AND EAX，若 EAX = 0 則 ZF = 1
                ; 慣用技巧：快速判斷暫存器是否為零
```

#### SF — Sign Flag（符號旗標）

運算結果的**最高位元（MSB）**的值，用於判斷有號數的正負。

```asm
mov al, 127     ; 0b01111111（正數，MSB = 0）
add al, 1       ; AL = 128 = 0b10000000（MSB = 1），SF = 1
                ; 有號數解讀：128 → -128（發生溢位！OF 也會被設定）

mov eax, -1     ; 0xFFFFFFFF（MSB = 1）
; SF = 1（表示結果為負數）

mov eax, 100    ; 正數（MSB = 0）
; SF = 0
```

#### OF — Overflow Flag（溢位旗標）

**有號數運算**超出該資料寬度的有號數範圍時置 1。

```
8-bit 有號數範圍：-128 ～ +127
16-bit 有號數範圍：-32768 ～ +32767
32-bit 有號數範圍：-2,147,483,648 ～ +2,147,483,647
```

```asm
; 正 + 正 = 負（溢位）
mov al, 127     ; 最大正數
add al, 1       ; AL = 128 → 有號解讀為 -128，OF = 1

; 負 + 負 = 正（溢位）
mov al, -128    ; 最小負數（0x80）
add al, -1      ; AL = 127，OF = 1（結果符號錯誤）

; 正 + 負 = 不會溢位
mov al, 100
add al, -50     ; AL = 50，OF = 0
```

**CF 與 OF 的差別：**

| 旗標 | 適用於 | 觸發條件 |
|------|--------|---------|
| CF | 無號數（Unsigned）| 結果超出無號數範圍（有進位/借位） |
| OF | 有號數（Signed） | 結果超出有號數範圍（符號位異常改變） |

同一個運算，CF 和 OF 可以獨立設定：

```asm
mov al, 0x80    ; AL = 128（無號），-128（有號）
add al, 0x80    ; = 0x100

; AL = 0x00（低 8 位）
; CF = 1（無號：256 超出 0～255 的範圍）
; OF = 1（有號：-128 + -128 = -256，超出 -128～127）
```

#### PF — Parity Flag（奇偶旗標）

運算結果**最低 8 位元**中，1 的個數為**偶數**時置 1。

```asm
mov al, 0b00000111  ; 1 的個數 = 3（奇數），PF = 0
mov al, 0b00001111  ; 1 的個數 = 4（偶數），PF = 1
mov al, 0b10101010  ; 1 的個數 = 4（偶數），PF = 1
```

> 奇偶旗標源自早期資料通訊的奇偶校驗需求，現代程式中較少直接使用。

#### AF — Auxiliary Carry Flag（輔助進位旗標）

第 3 位元（bit 3）向第 4 位元（bit 4）進位時置 1，主要用於 BCD（Binary Coded Decimal）運算。

```asm
mov al, 0x09    ; 0b00001001
add al, 0x01    ; 0b00001010
; bit 3 → bit 4 發生進位，AF = 1
; DAA 指令（Decimal Adjust after Addition）會用到 AF
```

#### DF — Direction Flag（方向旗標）

控制字串操作指令（MOVSB、CMPSB 等）的方向：

```asm
cld             ; Clear Direction Flag：DF = 0，字串操作向高位址前進（++）
std             ; Set Direction Flag：DF = 1，字串操作向低位址倒退（--）

; DF = 0（預設）
mov rsi, src
mov rdi, dst
mov rcx, 10
rep movsb       ; 從低位址到高位址複製（src++, dst++）

; DF = 1
std
rep movsb       ; 從高位址到低位址複製（src--, dst--）
; 通常需在完成後用 CLD 還原
```

## 3.6 旗標與條件跳躍指令

### 3.6.1 CMP 與 TEST 指令

在跳躍之前，通常先用 `CMP` 或 `TEST` 設定旗標：

```asm
; CMP dst, src：等同於 SUB dst, src，但不儲存結果，只更新旗標
cmp eax, ebx    ; 計算 EAX - EBX，設定 CF/ZF/SF/OF/PF/AF
cmp eax, 0      ; 檢查 EAX 是否為 0

; TEST dst, src：等同於 AND dst, src，但不儲存結果，只更新旗標
test eax, eax   ; EAX AND EAX，若 EAX=0 則 ZF=1（常用技巧）
test al, 0x01   ; 檢查 AL 最低位是否為 1（判斷奇偶）
test eax, 0x80000000 ; 檢查最高位（符號位）
```

**CMP 指令的旗標設定規律（以 `CMP EAX, EBX` 為例）：**

| 情況 | EAX - EBX 結果 | ZF | CF | SF | OF |
|------|--------------|----|----|----|----|
| EAX = EBX | 0 | 1 | 0 | 0 | 0 |
| EAX > EBX（無號）| 正 | 0 | 0 | — | — |
| EAX < EBX（無號）| 借位 | 0 | 1 | — | — |
| EAX > EBX（有號）| 正 | 0 | — | 0 | 0 |
| EAX < EBX（有號）| 負 | 0 | — | 1 | 0 |
| 有溢位 | — | — | — | — | 1 |

### 3.6.2 條件跳躍指令完整對照表

#### 基於單一旗標的跳躍

| 指令 | 全名 | 觸發條件 | 旗標 |
|------|------|---------|------|
| `JZ` / `JE` | Jump if Zero / Equal | ZF = 1 | ZF |
| `JNZ` / `JNE` | Jump if Not Zero / Not Equal | ZF = 0 | ZF |
| `JC` | Jump if Carry | CF = 1 | CF |
| `JNC` | Jump if No Carry | CF = 0 | CF |
| `JS` | Jump if Sign（負數）| SF = 1 | SF |
| `JNS` | Jump if No Sign（非負）| SF = 0 | SF |
| `JO` | Jump if Overflow | OF = 1 | OF |
| `JNO` | Jump if No Overflow | OF = 0 | OF |
| `JP` / `JPE` | Jump if Parity Even | PF = 1 | PF |
| `JNP` / `JPO` | Jump if Parity Odd | PF = 0 | PF |

#### 無號數比較後跳躍（After CMP，Unsigned）

| 指令 | 等效 | 條件 | 旗標組合 |
|------|------|------|---------|
| `JA` | Jump if Above（大於）| CF=0 且 ZF=0 | CF OR ZF = 0 |
| `JAE` / `JNB` | Jump if Above or Equal（大於等於）| CF=0 | CF = 0 |
| `JB` / `JNAE` | Jump if Below（小於）| CF=1 | CF = 1 |
| `JBE` / `JNA` | Jump if Below or Equal（小於等於）| CF=1 或 ZF=1 | CF OR ZF = 1 |

#### 有號數比較後跳躍（After CMP，Signed）

| 指令 | 等效 | 條件 | 旗標組合 |
|------|------|------|---------|
| `JG` | Jump if Greater（大於）| ZF=0 且 SF=OF | ZF=0 AND SF=OF |
| `JGE` | Jump if Greater or Equal（大於等於）| SF=OF | SF = OF |
| `JL` | Jump if Less（小於）| SF≠OF | SF ≠ OF |
| `JLE` | Jump if Less or Equal（小於等於）| ZF=1 或 SF≠OF | ZF=1 OR SF≠OF |

### 3.6.3 有號數 vs 無號數跳躍指令選擇

```asm
; 比較 EAX 和 EBX 後跳躍

; ── 情境一：EAX 和 EBX 是無號數（如 unsigned int）
cmp eax, ebx
ja  greater_unsigned    ; 無號大於：使用 JA/JB/JAE/JBE
jb  less_unsigned

; ── 情境二：EAX 和 EBX 是有號數（如 int）
cmp eax, ebx
jg  greater_signed      ; 有號大於：使用 JG/JL/JGE/JLE
jl  less_signed
```

**混用有號/無號跳躍指令是常見的錯誤來源！**

```asm
mov eax, -1             ; 有號 -1 = 無號 0xFFFFFFFF = 4294967295
mov ebx, 1
cmp eax, ebx

jg  wrong_label         ; 有號比較：-1 < 1，不跳（正確語意）
ja  wrong_label         ; 無號比較：4294967295 > 1，會跳！（錯誤語意）
```

## 3.7 旗標的讀取與操作

### 3.7.1 SETcc 指令：旗標值存入暫存器

`SETcc` 系列指令根據旗標條件，將 0 或 1 存入 8-bit 暫存器或記憶體。

```asm
cmp eax, ebx
sete  al        ; 若 ZF=1（EAX=EBX），AL = 1；否則 AL = 0
setg  bl        ; 若 JG 條件成立，BL = 1；否則 BL = 0
setl  [result]  ; 若 JL 條件成立，記憶體 result = 1；否則 = 0
```

**對應 C 語言的比較運算子：**

```c
// C 程式碼
int result = (a == b);
```

```asm
; 對應組語
cmp eax, ebx        ; a vs b
sete al             ; AL = (EAX == EBX) ? 1 : 0
movzx eax, al       ; zero-extend AL 到 EAX（清除高位）
```

### 3.7.2 LAHF / SAHF：旗標與 AH 互轉

```asm
LAHF    ; AH = RFLAGS 的低 8 位元（SF/ZF/AF/PF/CF）
SAHF    ; RFLAGS 的低 8 位元 = AH（還原旗標）
```

### 3.7.3 PUSHF / POPF：旗標入棧出棧

```asm
PUSHF   ; RFLAGS 的低 16 位元壓入堆疊
POPF    ; 從堆疊還原 FLAGS
PUSHFQ  ; 完整 64-bit RFLAGS 壓入堆疊（x86-64 用）
POPFQ   ; 從堆疊還原完整 64-bit RFLAGS
```

**常見用途：保存和還原旗標狀態**

```asm
pushfq              ; 保存目前旗標
; ... 執行可能修改旗標的操作 ...
popfq               ; 還原旗標
```

### 3.7.4 直接清除或設定特定旗標

```asm
; 清除旗標
CLC     ; Clear Carry Flag：CF = 0
CLD     ; Clear Direction Flag：DF = 0
CLI     ; Clear Interrupt Enable Flag：IF = 0（禁止外部中斷，需特權）

; 設定旗標
STC     ; Set Carry Flag：CF = 1
STD     ; Set Direction Flag：DF = 1
STI     ; Set Interrupt Enable Flag：IF = 1（允許外部中斷，需特權）

; 取反
CMC     ; Complement Carry：CF = NOT CF
```

## 3.8 實際應用範例

### 3.8.1 判斷一個數的正負與零

```asm
; 判斷 EAX 的正負零，並跳到對應標號
test eax, eax       ; 設定 ZF 和 SF（AND 不影響 CF/OF，比 CMP 乾淨）
jz   is_zero        ; ZF = 1 → EAX = 0
js   is_negative    ; SF = 1 → EAX 為負數（MSB = 1）
; 其餘情況：SF = 0 且 ZF = 0 → EAX 為正數
jmp  is_positive

is_zero:
    ; 處理 0 的情況
    jmp done
is_negative:
    ; 處理負數的情況
    jmp done
is_positive:
    ; 處理正數的情況
done:
```

### 3.8.2 多精度加法（使用 CF）

```asm
; 計算 128-bit 加法：A + B → 結果
; A 存在 RAX:RBX（RAX 為高 64 位）
; B 存在 RCX:RDX（RCX 為高 64 位）

add rbx, rdx    ; 低 64 位相加，CF 記錄進位
adc rax, rcx    ; 高 64 位相加 + 進位（ADC = Add with Carry）
```

### 3.8.3 計算絕對值（有號數）

```asm
; 計算 EAX 的絕對值（不使用分支）
mov  ebx, eax
sar  ebx, 31    ; 算術右移 31 位：若為負數，EBX = 0xFFFFFFFF；正數 = 0
xor  eax, ebx   ; 若為負：EAX = ~EAX；若為正：EAX 不變
sub  eax, ebx   ; 若為負：EAX = ~EAX + 1 = -原值；若為正：EAX - 0 不變
```

### 3.8.4 判斷奇偶數

```asm
; 判斷 EAX 是否為偶數
test eax, 1         ; 檢查最低位（bit 0）
jz   is_even        ; ZF = 1 → bit 0 = 0 → 偶數
; 否則為奇數

; 或者：使用 PF 的方式（對最低 byte 的奇偶）
; 注意：PF 是結果的低 8 位元中 1 的個數的奇偶，
;       與「數字本身的奇偶」是不同的概念！
```

## 3.9 浮點數與 SIMD 暫存器簡介

本節僅介紹基本概念，詳細使用請參考進階資料。

### 3.9.1 x87 FPU 暫存器（ST0 ～ ST7）

x87 FPU 使用一個 8 個暫存器的**堆疊結構**：

```
ST0  ← 堆疊頂端（浮點運算的主要暫存器）
ST1
ST2
...
ST7
```

```asm
; 載入浮點數到 FPU 堆疊
fld  dword [pi]     ; 將 [pi] 的值 push 到 FPU 堆疊（ST0 = [pi]）
fld  dword [r]      ; ST0 = [r]，原來的 ST0 變成 ST1

; 浮點運算
fmul st0, st0       ; ST0 = ST0 * ST0（r 的平方）
fmul dword [pi]     ; ST0 = ST0 * pi
fadd st0, st0       ; ST0 = ST0 * 2（模擬 2πr² 的一半... 僅示意）

; 儲存結果
fstp dword [result] ; pop ST0 並存到 [result]
```

### 3.9.2 SSE / AVX 暫存器

| 暫存器組 | 大小 | 指令集 | 可容納資料 |
|---------|------|--------|-----------|
| XMM0～XMM15 | 128-bit | SSE/SSE2 | 4 個 float，或 2 個 double |
| YMM0～YMM15 | 256-bit | AVX/AVX2 | 8 個 float，或 4 個 double |
| ZMM0～ZMM31 | 512-bit | AVX-512 | 16 個 float，或 8 個 double |

```asm
; SSE：一次處理 4 個浮點數（SIMD）
movaps xmm0, [arr_a]    ; 載入 4 個 float
movaps xmm1, [arr_b]    ; 載入 4 個 float
addps  xmm0, xmm1       ; 同時計算 4 組加法
movaps [result], xmm0   ; 儲存 4 個結果
```

## 3.10 本章小結

**暫存器基礎：**
- 暫存器是 CPU 內部最快的儲存單元，所有運算都在暫存器中進行
- x86-64 有 16 個 64-bit 通用暫存器，每個都可存取其 32/16/8-bit 子部分
- 寫入 32-bit 名稱（如 EAX）會清除同暫存器的高 32 位元（zero-extend）

**特殊暫存器：**
- `RSP`：始終指向堆疊頂端，不可隨意使用
- `RBP`：用於建立堆疊框架，存取區域變數
- `RIP`：程式計數器，指向下一條指令，只能透過跳躍/呼叫間接修改

**旗標暫存器：**
- CF（進位）、ZF（零）、SF（符號）、OF（溢位）、PF（奇偶）、AF（輔助進位）、DF（方向）
- `CMP` = 做減法但不存結果；`TEST` = 做 AND 但不存結果，兩者都只更新旗標
- 無號數比較用 JA/JB/JAE/JBE；有號數比較用 JG/JL/JGE/JLE；混用會產生難以察覺的錯誤

**函式呼叫慣例（Linux System V AMD64 ABI）：**
- 前 6 個整數參數：RDI, RSI, RDX, RCX, R8, R9
- 回傳值：RAX
- Callee-saved：RBX, RBP, R12～R15
