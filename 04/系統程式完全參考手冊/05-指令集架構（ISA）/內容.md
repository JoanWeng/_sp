# 第 5 章　指令集架構（ISA）

> **本章目標：** 全面掌握 x86-64 的指令集，涵蓋資料傳送、算術運算、邏輯與位元操作、控制流程、字串操作等各類指令的語法、語意、旗標影響與實際使用情境。

---

## 5.1 指令集架構概述

### 5.1.1 什麼是 ISA？

**指令集架構（Instruction Set Architecture, ISA）** 是 CPU 硬體與軟體之間的介面規範，定義了：

- CPU 支援哪些指令（指令集）
- 每條指令的格式（opcode、運算元數量與類型）
- 每條指令的語意（執行後的狀態變化）
- 暫存器的種類與數量
- 記憶體的存取方式
- 例外與中斷的處理方式

### 5.1.2 CISC vs RISC

x86-64 屬於 **CISC（Complex Instruction Set Computer）** 架構：

| 比較項目 | CISC（x86-64）| RISC（ARM、RISC-V）|
|---------|-------------|-------------------|
| 指令數量 | 數千條 | 數十至數百條 |
| 指令長度 | 可變（1～15 bytes）| 固定（通常 4 bytes）|
| 記憶體存取 | 可在算術指令中直接存取 | 限 Load/Store 指令 |
| 解碼複雜度 | 高 | 低 |
| 程式密度 | 高（相同功能需要較少的指令）| 低 |
| 常見場景 | 桌機、伺服器 | 行動裝置、嵌入式 |

### 5.1.3 指令的基本格式

```
[前綴]  助記符  [目的運算元]  [, 來源運算元]  [; 註解]

前綴（Prefix）：可選，如 REP、LOCK、REX
助記符：指令名稱
運算元：0 到 2 個（少數指令有 3 個，如 IMUL、SHLD）
```

指令依運算元數量分類：

```asm
; 零運算元
NOP                 ; 無操作
RET                 ; 返回
SYSCALL             ; 系統呼叫

; 單一運算元
INC EAX             ; EAX++
PUSH RBX            ; 壓堆疊
NOT EAX             ; 位元 NOT

; 雙運算元（最常見）
MOV EAX, EBX        ; 目的 ← 來源
ADD EAX, 10         ; 目的 += 來源

; 三運算元（較少）
IMUL EAX, EBX, 5    ; EAX = EBX × 5
SHLD EAX, EBX, 4    ; 雙精度左移
```

## 5.2 資料傳送指令

### 5.2.1 MOV — 基本資料傳送

`MOV` 是最常用的指令，將來源的值複製到目的地。

```
MOV dst, src
```

```asm
; 暫存器 ← 立即值
MOV EAX, 42
MOV RAX, 0xFFFFFFFFFFFFFFFF

; 暫存器 ← 暫存器
MOV EAX, EBX
MOV RAX, RCX

; 暫存器 ← 記憶體
MOV EAX, [var]
MOV RAX, [RBX + 8]

; 記憶體 ← 暫存器
MOV [var], EAX
MOV [RBX + RCX*4], EDX

; 記憶體 ← 立即值（需指定大小）
MOV DWORD [counter], 0
MOV BYTE  [flag], 1
```

**MOV 的限制：**
```asm
MOV [mem1], [mem2]   ; ✗ 兩個不能都是記憶體
MOV EAX, RBX         ; ✗ 大小不同
MOV CS, EAX          ; ✗ 不能直接寫入 CS 區段暫存器
```

### 5.2.2 零擴展與符號擴展傳送

當從較小的來源移動到較大的目的時，需要決定如何填充高位元：

#### MOVZX — Move with Zero-Extend（零擴展）

```asm
MOVZX EAX, BL       ; EAX = 零擴展的 BL（高位填 0）
MOVZX EAX, WORD [mem]; EAX = 零擴展的 16-bit 記憶體值
MOVZX RAX, BL       ; RAX = 零擴展的 BL 到 64-bit

; 範例：BL = 0xFF（255 無號，或 -1 有號）
; MOVZX EAX, BL → EAX = 0x000000FF = 255（無號解讀）
```

#### MOVSX — Move with Sign-Extend（符號擴展）

```asm
MOVSX EAX, BL       ; EAX = 符號擴展的 BL（以符號位填充高位）
MOVSX RAX, WORD [mem]

; 範例：BL = 0xFF（-1 有號）
; MOVSX EAX, BL → EAX = 0xFFFFFFFF = -1（有號解讀）
```

#### MOVSXD — 32-bit 符號擴展到 64-bit

```asm
MOVSXD RAX, EBX     ; RAX = 符號擴展的 EBX（32→64 bit）
; 注意：在 x86-64，寫入 EBX 自動 zero-extend 到 RBX，
;       所以 MOVSXD 主要用於有號數需要符號擴展的場合
```

**零擴展 vs 符號擴展的選擇：**

```c
// C 語言對應
unsigned char uc = 0xFF;    // 255（無號）
signed   char sc = 0xFF;    // -1（有號）

int a = (int)(unsigned int)uc;  // = 255，用 MOVZX
int b = (int)sc;                // = -1，用 MOVSX
```

### 5.2.3 XCHG — 交換

```asm
XCHG EAX, EBX       ; 交換 EAX 和 EBX（不需要暫存器中轉）
XCHG EAX, [mem]     ; 交換暫存器和記憶體（隱含 LOCK 前綴，不可分割）
```

> `XCHG` 與記憶體操作時自動加上 `LOCK` 前綴，保證原子性，常用於自旋鎖（Spinlock）實作。

### 5.2.4 PUSH / POP — 堆疊操作

```asm
PUSH src             ; RSP -= 8，[RSP] = src（x86-64 預設 64-bit）
POP  dst             ; dst = [RSP]，RSP += 8

; 可以 PUSH/POP 的對象
PUSH RAX             ; 暫存器
PUSH 42              ; 立即值
PUSH [mem]           ; 記憶體（讀取後壓入）
POP  RBX
POP  [mem]           ; 彈出到記憶體
```

### 5.2.5 LEA — 載入有效位址

```asm
LEA dst, [expr]      ; dst = expr 計算的位址（不存取記憶體）

LEA RDI, [var]       ; RDI = var 的位址（&var）
LEA EAX, [EBX*5]     ; EAX = EBX × 5（利用 SIB 做快速乘法）
LEA RAX, [RIP + msg] ; RIP 相對定址（PIC）
```

### 5.2.6 CMOVcc — 條件傳送

根據旗標狀態決定是否執行 `MOV`，可以避免分支（提升 pipeline 效率）：

```asm
CMP EAX, EBX
CMOVG  ECX, EDX      ; 若 EAX > EBX（有號），ECX = EDX
CMOVL  ECX, EDX      ; 若 EAX < EBX（有號），ECX = EDX
CMOVE  ECX, EDX      ; 若 EAX = EBX，ECX = EDX
CMOVNE ECX, EDX      ; 若 EAX ≠ EBX，ECX = EDX

; 實際應用：求最大值（無分支版本）
; if (EBX > EAX) EAX = EBX
CMP EAX, EBX
CMOVL EAX, EBX       ; 若 EAX < EBX，EAX = EBX
```

所有條件跳躍的條件（JE/JNE/JG/JL 等）都有對應的 CMOVcc 版本。

## 5.3 算術運算指令

### 5.3.1 加法指令

#### ADD — 加法

```asm
ADD dst, src         ; dst = dst + src，影響 CF/ZF/SF/OF/PF/AF

ADD EAX, EBX         ; EAX += EBX
ADD EAX, 100         ; EAX += 100
ADD [counter], 1     ; 記憶體 += 1（需指定大小）
ADD DWORD [counter], 1
```

#### ADC — 帶進位加法（Add with Carry）

```asm
ADC dst, src         ; dst = dst + src + CF

; 用途：多精度加法（超過 64-bit 的整數）
; 計算 128-bit 加法：(RDX:RAX) + (RCX:RBX)
ADD RAX, RBX         ; 低 64 位元相加，CF = 進位
ADC RDX, RCX         ; 高 64 位元 + 進位
```

#### INC — 遞增

```asm
INC EAX              ; EAX++
INC DWORD [counter]  ; 記憶體值++
```

> `INC` 不影響 `CF`，這是與 `ADD reg, 1` 的差異。  
> 在多精度計算中，使用 `ADD reg, 1` + `ADC` 比 `INC` + `ADC` 更安全，因為 `INC` 不更新 `CF`。

### 5.3.2 減法指令

#### SUB — 減法

```asm
SUB dst, src         ; dst = dst - src，影響 CF/ZF/SF/OF/PF/AF

SUB EAX, EBX         ; EAX -= EBX
SUB EAX, 1           ; EAX -= 1
SUB [var], ECX
```

#### SBB — 帶借位減法（Subtract with Borrow）

```asm
SBB dst, src         ; dst = dst - src - CF

; 128-bit 減法：(RDX:RAX) - (RCX:RBX)
SUB RAX, RBX         ; 低位相減，CF = 借位
SBB RDX, RCX         ; 高位 - RCX - 借位
```

#### DEC — 遞減

```asm
DEC EAX              ; EAX--（不影響 CF）
DEC BYTE [flag]
```

#### NEG — 取負（二補數）

```asm
NEG EAX              ; EAX = -EAX = ~EAX + 1
; 若 EAX = 5  → EAX = -5
; 若 EAX = 0  → EAX = 0，CF = 0
; 若 EAX ≠ 0 → CF = 1
```

### 5.3.3 乘法指令

#### MUL — 無號乘法

```asm
MUL src              ; 單運算元，隱含使用 AL/AX/EAX/RAX

; 8-bit：AX = AL × src8
MUL BL               ; AX = AL × BL

; 16-bit：DX:AX = AX × src16
MUL BX               ; DX:AX = AX × BX

; 32-bit：EDX:EAX = EAX × src32
MUL EBX              ; EDX:EAX = EAX × EBX

; 64-bit：RDX:RAX = RAX × src64
MUL RBX              ; RDX:RAX = RAX × RBX
```

**為什麼結果是雙倍寬度？**

兩個 N-bit 數相乘，結果最大為 2N-bit。例如 0xFF × 0xFF = 0xFE01，需要 16-bit 才能表示。

#### IMUL — 有號乘法（三種形式）

```asm
; 形式 1：單運算元（與 MUL 類似，有號版本）
IMUL EBX             ; EDX:EAX = EAX × EBX（有號）

; 形式 2：雙運算元（截斷到相同大小）
IMUL EAX, EBX        ; EAX = EAX × EBX（結果截斷為 32-bit）
IMUL RCX, RDX        ; RCX = RCX × RDX

; 形式 3：三運算元（最常用）
IMUL EAX, EBX, 10   ; EAX = EBX × 10（不改 EBX）
IMUL RCX, RDX, 100  ; RCX = RDX × 100

; 常見用法：計算陣列偏移
IMUL RCX, RBX, 12   ; RCX = 索引 × sizeof(struct)（12 bytes）
```

### 5.3.4 除法指令

#### DIV — 無號除法

```asm
DIV src              ; 被除數在 AX / DX:AX / EDX:EAX / RDX:RAX

; 32-bit：
; 被除數：EDX:EAX（64-bit）
; 除數：src（32-bit）
; 商：EAX，餘數：EDX

MOV EAX, 1000
MOV EDX, 0          ; 清除高位（若 EDX 有值會影響結果！）
MOV ECX, 7
DIV ECX              ; EAX = 1000 / 7 = 142，EDX = 1000 % 7 = 6
```

> ⚠️ **除法陷阱：** 若商超過目的暫存器的範圍（如 EDX:EAX = 很大的數 ÷ 很小的除數），會觸發 **#DE（Division Exception）**，導致程式崩潰。使用前應確認被除數的高位是否正確清零。

#### IDIV — 有號除法

```asm
; 使用前需符號擴展被除數
MOV EAX, -100       ; 被除數
CDQ                 ; 符號擴展 EAX 到 EDX:EAX（Convert Dword to Qword）
MOV ECX, 7
IDIV ECX            ; EAX = -100 / 7 = -14，EDX = -100 % 7 = -2
```

**符號擴展輔助指令：**

```asm
CBW                 ; AX = 符號擴展 AL（Convert Byte to Word）
CWD                 ; DX:AX = 符號擴展 AX（Convert Word to Dword）
CDQ                 ; EDX:EAX = 符號擴展 EAX（Convert Dword to Qword）
CQO                 ; RDX:RAX = 符號擴展 RAX（Convert Qword to Oword）
```

## 5.4 邏輯與位元運算指令

### 5.4.1 AND — 位元 AND

```asm
AND dst, src         ; dst = dst AND src，CF=0, OF=0，更新 ZF/SF/PF

AND EAX, 0x0F        ; 保留低 4 位元（清除高 28 位）
AND EAX, 0xFFFFFF00  ; 清除低 8 位元
AND EAX, EBX         ; 暫存器間 AND
AND BYTE [flag], 0xFE; 清除最低位（偶數對齊）

; 常見用途：遮罩（Masking）
MOV EAX, 0b10110101
AND EAX, 0b00001111  ; EAX = 0b00000101（只保留低 4 位）
```

### 5.4.2 OR — 位元 OR

```asm
OR dst, src          ; dst = dst OR src，CF=0, OF=0，更新 ZF/SF/PF

OR EAX, 0x80000000   ; 設定最高位
OR EAX, 0x20         ; 設定 bit 5（小寫化：大寫 ASCII OR 0x20 = 小寫）
OR EAX, EBX

; 常見用途：設定特定位元
MOV EAX, 0b10100000
OR  EAX, 0b00001111  ; EAX = 0b10101111（設定低 4 位）
```

### 5.4.3 XOR — 位元 XOR

```asm
XOR dst, src         ; dst = dst XOR src，CF=0, OF=0，更新 ZF/SF/PF

XOR EAX, EAX         ; EAX = 0（最有效率的清零方式，比 MOV EAX,0 短）
XOR EAX, 0xFF        ; 取反低 8 位元
XOR EAX, EBX

; 常見用途 1：清零（最慣用手法）
XOR RAX, RAX         ; RAX = 0

; 常見用途 2：切換位元（Toggle）
XOR EAX, 0x20        ; 切換 bit 5（大小寫互換）

; 常見用途 3：不用暫存器交換兩個值
XOR EAX, EBX         ; EAX = EAX XOR EBX
XOR EBX, EAX         ; EBX = EBX XOR (EAX XOR EBX) = 原始 EAX
XOR EAX, EBX         ; EAX = (EAX XOR EBX) XOR 原始EAX = 原始 EBX
```

### 5.4.4 NOT — 位元 NOT（取反）

```asm
NOT dst              ; dst = ~dst（位元全部反轉），不影響任何旗標

NOT EAX              ; 所有位元取反
NOT BYTE [mask]
```

### 5.4.5 TEST — 測試位元（不儲存結果的 AND）

```asm
TEST dst, src        ; 計算 dst AND src，設定旗標但不儲存結果

TEST EAX, EAX        ; 常用：快速測試 EAX 是否為 0（比 CMP EAX,0 短）
TEST AL, 1           ; 測試最低位是否為 1（判斷奇偶）
TEST EAX, 0x80000000 ; 測試最高位（符號位）

; 範例：判斷整數奇偶
TEST EAX, 1
JZ   is_even         ; ZF=1 表示最低位為 0，即偶數
; 否則為奇數
```

## 5.5 移位與旋轉指令

### 5.5.1 SHL / SAL — 邏輯/算術左移

```asm
SHL dst, count       ; 左移 count 位，右邊補 0，最左位移入 CF
SAL dst, count       ; 與 SHL 完全相同（算術左移等同邏輯左移）

SHL EAX, 1           ; EAX × 2
SHL EAX, 3           ; EAX × 8（等同 EAX × 2³）
SHL EAX, CL          ; 移位量由 CL 決定（動態移位）

; 範例：
MOV EAX, 1
SHL EAX, 4           ; EAX = 16（1 << 4）
```

**左移 = 乘以 2 的冪次：**

```
SHL n 位 = × 2ⁿ（適用於無號數和有號數，但要注意溢位）
```

### 5.5.2 SHR — 邏輯右移

```asm
SHR dst, count       ; 右移 count 位，左邊補 0，最右位移入 CF

SHR EAX, 1           ; EAX ÷ 2（無號）
SHR EAX, 2           ; EAX ÷ 4
SHR EAX, CL          ; 動態移位

; 用於無號整數除以 2 的冪次
MOV EAX, 100
SHR EAX, 2           ; EAX = 25（100 ÷ 4）
```

### 5.5.3 SAR — 算術右移（符號擴展）

```asm
SAR dst, count       ; 右移 count 位，左邊補符號位（MSB），最右位移入 CF

; 用於有號整數除以 2 的冪次（保留符號）
MOV EAX, -100
SAR EAX, 2           ; EAX = -25（-100 ÷ 4，向負無窮大取整）

MOV EAX, -1          ; 0xFFFFFFFF
SAR EAX, 1           ; EAX = -1（0xFFFFFFFF，符號位補 1）

; SHR vs SAR 對負數的差異：
MOV EAX, -8          ; 0xFFFFFFF8
SHR EAX, 1           ; EAX = 0x7FFFFFFC = 2147483644（錯誤！）
SAR EAX, 1           ; EAX = -4 = 0xFFFFFFFC（正確！）
```

### 5.5.4 ROL / ROR — 旋轉

```asm
ROL dst, count       ; 左旋轉：移出的位元從右邊補回，同時進 CF
ROR dst, count       ; 右旋轉：移出的位元從左邊補回，同時進 CF

MOV AL, 0b10110001
ROL AL, 1            ; AL = 0b01100011，CF = 1（最高位旋出）
ROR AL, 1            ; AL = 0b10110001，CF = 1（最低位旋出，回復原值）

; 旋轉不丟失位元，常用於雜湊函式和密碼學
```

### 5.5.5 RCL / RCR — 帶進位旋轉

```asm
RCL dst, count       ; 左旋轉（包含 CF 形成 N+1 位元的環）
RCR dst, count       ; 右旋轉（包含 CF 形成 N+1 位元的環）

; CF 也參與旋轉，形成一個更大的環
; 用於多精度移位操作
```

### 5.5.6 SHLD / SHRD — 雙精度移位

```asm
SHLD dst, src, count ; dst 左移，從 src 的高位補入（dst 和 src 不變）
SHRD dst, src, count ; dst 右移，從 src 的低位補入

; 範例：64-bit 值存在 EDX:EAX，左移 4 位
SHLD EDX, EAX, 4     ; EDX 左移 4，從 EAX 的高 4 位補入
SHL  EAX, 4          ; EAX 左移 4（低位自然補 0）
```

## 5.6 比較與控制流程指令

### 5.6.1 CMP — 比較

```asm
CMP dst, src         ; 計算 dst - src，設定旗標，不儲存結果

CMP EAX, 0           ; 檢查 EAX 是否為 0
CMP EAX, EBX         ; 比較兩個暫存器
CMP DWORD [var], 100 ; 比較記憶體值與常數
```

### 5.6.2 無條件跳躍

```asm
JMP label            ; 直接跳到 label
JMP RBX              ; 跳到 RBX 存放的位址（間接跳躍）
JMP [RBX]            ; 跳到 RBX 所指位址存放的位址（跳躍表）
```

**短跳與近跳的機器碼差異：**

```asm
JMP SHORT label      ; 短跳（±127 bytes），機器碼：EB disp8
JMP NEAR  label      ; 近跳（±2GB），機器碼：E9 disp32
; 組譯器通常自動選擇最短的形式
```

### 5.6.3 條件跳躍完整分類

```asm
; ── 基於 ZF ──────────────────────────────
JE  / JZ   label     ; ZF=1（等於 / 為零）
JNE / JNZ  label     ; ZF=0（不等於 / 不為零）

; ── 基於 CF ──────────────────────────────
JC  / JB   label     ; CF=1（有進位 / 無號小於）
JNC / JAE  label     ; CF=0（無進位 / 無號大於等於）

; ── 無號數比較（after CMP）──────────────
JA  / JNBE label     ; CF=0 且 ZF=0（無號大於）
JAE / JNB  label     ; CF=0（無號大於等於）
JB  / JNAE label     ; CF=1（無號小於）
JBE / JNA  label     ; CF=1 或 ZF=1（無號小於等於）

; ── 有號數比較（after CMP）──────────────
JG  / JNLE label     ; ZF=0 且 SF=OF（有號大於）
JGE / JNL  label     ; SF=OF（有號大於等於）
JL  / JNGE label     ; SF≠OF（有號小於）
JLE / JNG  label     ; ZF=1 或 SF≠OF（有號小於等於）

; ── 基於 SF / OF ──────────────────────────
JS         label     ; SF=1（結果為負）
JNS        label     ; SF=0（結果非負）
JO         label     ; OF=1（有溢位）
JNO        label     ; OF=0（無溢位）

; ── 基於 PF ──────────────────────────────
JP  / JPE  label     ; PF=1（奇偶偶數）
JNP / JPO  label     ; PF=0（奇偶奇數）
```

### 5.6.4 CALL / RET — 副程式呼叫

```asm
CALL label           ; PUSH RIP（返回位址），JMP label
CALL RBX             ; 間接呼叫（函式指標）
CALL [table + RCX*8] ; 透過跳躍表呼叫（虛擬函式）

RET                  ; POP RIP（彈出返回位址並跳回）
RET 8                ; POP RIP，RSP += 8（清理堆疊參數）
```

### 5.6.5 LOOP — 計數迴圈

```asm
LOOP label           ; ECX--（或 RCX--）；若 ≠ 0 則跳回 label
LOOPE  label         ; ECX-- 且 ZF=1 則跳（Loop while Equal）
LOOPNE label         ; ECX-- 且 ZF=0 則跳（Loop while Not Equal）

; 等同於：
; DEC ECX
; JNZ label

; 範例：迴圈 10 次
MOV ECX, 10
.loop:
    ; ... 迴圈體 ...
    LOOP .loop       ; ECX-- 後若不為 0 則繼續
```

> ⚠️ `LOOP` 在現代 CPU 上效能並不優於 `DEC + JNZ`，且固定使用 ECX/RCX，彈性較差，實際程式碼中較少使用。

## 5.7 字串操作指令

字串指令以 **RSI**（來源）和 **RDI**（目的）為隱含運算元，並根據 **DF** 旗標決定方向。

### 5.7.1 基本字串指令

```asm
; MOVS — 複製
MOVSB                ; [RDI] = [RSI]，RSI 和 RDI 各 ±1
MOVSW                ; 複製 2 bytes，RSI/RDI ±2
MOVSD                ; 複製 4 bytes，RSI/RDI ±4
MOVSQ                ; 複製 8 bytes，RSI/RDI ±8

; CMPS — 比較
CMPSB                ; [RSI] - [RDI]，設定旗標，RSI/RDI ±1
CMPSW / CMPSD / CMPSQ

; SCAS — 掃描（在記憶體中搜尋值）
SCASB                ; AL - [RDI]，設定旗標，RDI ±1
SCASW / SCASD / SCASQ

; STOS — 儲存（填充記憶體）
STOSB                ; [RDI] = AL，RDI ±1
STOSW / STOSD / STOSQ

; LODS — 載入
LODSB                ; AL = [RSI]，RSI ±1
LODSW / LODSD / LODSQ
```

### 5.7.2 REP 前綴

`REP` 前綴使字串指令重複執行 RCX 次：

```asm
REP   MOVSB          ; 重複 RCX 次：[RDI] = [RSI]，RDI++，RSI++，RCX--
REPE  CMPSB          ; 重複直到 ZF=0 或 RCX=0（比較相等則繼續）
REPNE CMPSB          ; 重複直到 ZF=1 或 RCX=0（比較不等則繼續）
REPNE SCASB          ; 搜尋：直到找到 AL 所指的字元或 RCX=0

; memcpy(dst, src, n) 的簡易實作：
; 設定：RDI = dst，RSI = src，RCX = n
CLD                  ; DF = 0（正向）
REP MOVSB

; memset(dst, 0, n) 的簡易實作：
; 設定：RDI = dst，RAX = 0（填充值），RCX = n/8
CLD
REP STOSQ            ; 每次填充 8 bytes（用 STOSB 則逐 byte）

; strlen（找字串長度）：
; RDI = 字串，AL = 0（找 null）
XOR  AL, AL          ; AL = 0
MOV  RCX, -1         ; 最大搜尋長度（無符號）
REPNE SCASB          ; 搜尋 AL 在 [RDI] 中的位置
NOT  RCX             ; 長度 = NOT(RCX) - 1（因 RCX 從 -1 開始遞減）
DEC  RCX
```

## 5.8 其他重要指令

### 5.8.1 NOP — 無操作

```asm
NOP                  ; 機器碼：0x90，消耗一個時脈但不做任何事

; 用途 1：占位（留下空間供後續修改）
; 用途 2：對齊（讓後續指令對齊到 16-byte 邊界，提升 fetch 效率）
; 用途 3：延遲（早期硬體中用於時序控制）
NOP
NOP
NOP                  ; 三個 NOP = 3 個時脈的延遲

; 多 byte NOP（NASM）
DB 0x0F, 0x1F, 0x00  ; 3-byte NOP（更有效率的對齊）
```

### 5.8.2 XCHG / BSWAP — 位元組序轉換

```asm
BSWAP EAX            ; 反轉 EAX 的位元組順序（大端 ↔ 小端）
; EAX = 0x12345678 → EAX = 0x78563412

BSWAP RAX            ; 64-bit 版本

; 用途：網路協定（網路位元組序是大端，x86 是小端）
; 相當於 C 的 ntohl()
```

### 5.8.3 SETcc — 條件設定

```asm
SETE  AL             ; 若 ZF=1，AL = 1；否則 AL = 0
SETG  BL             ; 若 JG 條件，BL = 1；否則 BL = 0
SETL  [result]       ; 若 JL 條件，記憶體 = 1；否則 = 0

; 實際應用：C 比較運算子的實現
; int result = (a > b);
CMP EAX, EBX
SETG AL
MOVZX EAX, AL        ; EAX = 0 或 1
```

### 5.8.4 BT / BTS / BTR / BTC — 位元測試與操作

```asm
BT  EAX, 3           ; 測試 EAX 的 bit 3，結果存入 CF
BTS EAX, 3           ; 測試 bit 3（→CF）並設定為 1
BTR EAX, 3           ; 測試 bit 3（→CF）並清除為 0
BTC EAX, 3           ; 測試 bit 3（→CF）並取反

; 應用：旗標陣列（Bit Array）
BTS [bitmap], ECX    ; 設定 bitmap 的第 ECX 個 bit
BTR [bitmap], ECX    ; 清除 bitmap 的第 ECX 個 bit
BT  [bitmap], ECX    ; 測試 bitmap 的第 ECX 個 bit，結果在 CF
JC  bit_was_set
```

### 5.8.5 BSF / BSR — 位元掃描

```asm
BSF EAX, EBX         ; Bit Scan Forward：找 EBX 最低的 1 位，結果（位元索引）存 EAX
BSR EAX, EBX         ; Bit Scan Reverse：找 EBX 最高的 1 位，結果存 EAX
; 若 EBX = 0，ZF = 1，EAX 未定義

; 應用：計算 log₂（最高位元位置）
MOV EBX, 64          ; 0b01000000
BSR EAX, EBX         ; EAX = 6（bit 6 是最高的 1 位）
; 等同於 floor(log₂(64)) = 6
```

### 5.8.6 CPUID — 查詢 CPU 特性

```asm
MOV EAX, 0           ; EAX = 查詢功能編號
CPUID                ; 結果存在 EAX/EBX/ECX/EDX
; EBX:EDX:ECX 包含 CPU 廠商字串（如 "GenuineIntel"）

MOV EAX, 1
CPUID                ; EAX = Family/Model/Stepping，ECX/EDX = Feature Flags
; 可用於判斷 SSE、AVX 等擴展指令是否可用
```

## 5.9 前綴（Prefix）

前綴是加在指令之前的 1-byte 修飾符，改變指令的行為。

### 5.9.1 REP / REPE / REPNE

```asm
REP    MOVSB         ; 重複 RCX 次
REPE   CMPSB         ; 重複直到 ZF=0（Equal）
REPNE  SCASB         ; 重複直到 ZF=1（Not Equal）
```

### 5.9.2 LOCK — 原子操作

```asm
LOCK ADD [counter], 1    ; 原子性地讓 counter 加 1（多執行緒安全）
LOCK XCHG EAX, [lock]   ; 原子交換（實作互斥鎖）
LOCK CMPXCHG [ptr], ECX  ; 原子比較交換（CAS 操作）
```

`LOCK` 前綴確保記憶體操作是**不可分割的**，防止多 CPU 同時存取同一記憶體位址時產生競態條件（Race Condition）。

### 5.9.3 REX — 64-bit 擴展

```asm
; REX 前綴（0x40～0x4F）由組譯器自動加入，無需手動撰寫
; 用於：存取 R8～R15、使用 64-bit 操作、存取 SPL/BPL/SIL/DIL
MOV R8, 100          ; 組譯器自動加 REX.R 位元
MOV RAX, RBX         ; 組譯器自動加 REX.W 位元
```

### 5.9.4 段前綴 — 指定記憶體區段

```asm
MOV EAX, FS:[0]      ; 從 FS 段基底 + 0 讀取（Thread Local Storage）
MOV EAX, GS:[28]     ; 從 GS 段基底 + 28 讀取（Linux kernel percpu）
```

## 5.10 指令對旗標的影響彙整

| 指令 | CF | ZF | SF | OF | PF | AF |
|------|----|----|----|----|----|----|
| `ADD` | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `ADC` | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `SUB / CMP` | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `INC / DEC` | — | ✓ | ✓ | ✓ | ✓ | ✓ |
| `NEG` | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `MUL` | ✓ | — | — | ✓ | — | — |
| `IMUL` | ✓ | — | — | ✓ | — | — |
| `AND / OR / XOR / TEST` | 0 | ✓ | ✓ | 0 | ✓ | — |
| `NOT` | — | — | — | — | — | — |
| `SHL / SHR / SAR` | ✓ | ✓ | ✓ | ✓* | ✓ | — |
| `ROL / ROR` | ✓ | — | — | ✓* | — | — |
| `MOV / LEA` | — | — | — | — | — | — |
| `PUSH / POP` | — | — | — | — | — | — |

> ✓：更新；—：不影響；0：強制清零；✓*：只在移位 1 位時有定義

## 5.11 本章小結

**資料傳送類：**
- `MOV`：最基本的複製指令，不能記憶體→記憶體
- `MOVZX` / `MOVSX`：跨大小傳送，決定以零擴展還是符號擴展填充高位
- `CMOVcc`：條件傳送，可消除分支提升效率

**算術類：**
- `ADD/SUB`：基本加減，影響全部算術旗標
- `ADC/SBB`：帶進位的加減法，用於多精度運算
- `INC/DEC`：不影響 CF（與 `ADD/SUB 1` 的差異）
- `MUL/IMUL`：無號/有號乘法，結果為雙倍寬度（MUL）或截斷（IMUL 雙/三運算元）
- `DIV/IDIV`：使用前必須正確設定被除數的高位（清零或符號擴展）

**邏輯與位元類：**
- `AND/OR/XOR/NOT`：位元操作；XOR 自身清零是常用慣用法
- `TEST`：不儲存的 AND，只設旗標，比 `CMP reg, 0` 更高效測試是否為零
- `SHL/SHR/SAR`：左移 = ×2ⁿ；SHR 用於無號除以 2ⁿ，SAR 用於有號

**控制流程類：**
- `JMP`：無條件跳躍
- `Jcc`：條件跳躍，無號數用 JA/JB，有號數用 JG/JL
- `CALL/RET`：函式呼叫與返回，透過堆疊傳遞返回位址

**字串類：**
- 隱含使用 RSI（來源）、RDI（目的）、RCX（計數）
- `REP MOVSB/STOSD/SCASB` 是簡潔的記憶體操作方式
