# 第 6 章　堆疊與副程式

> **本章目標：** 深入理解堆疊的結構與操作原理、副程式的呼叫與返回機制、堆疊框架的建立與銷毀、參數傳遞的各種方式，以及遞迴函式在記憶體中的實際行為。

---

## 6.1 堆疊的基本概念

### 6.1.1 什麼是堆疊？

**堆疊（Stack）** 是一種後進先出（LIFO，Last In First Out）的資料結構。在 x86-64 中，堆疊是一塊由作業系統配置的連續記憶體區域，有以下特性：

- **向低位址方向成長**：PUSH 時 RSP 減小，POP 時 RSP 增大
- **RSP 始終指向堆疊頂端**（最後一個被 PUSH 的資料）
- **每次 PUSH/POP 移動 8 bytes**（x86-64 預設操作單位）

```
記憶體位址（高 → 低）

0x7FFF0020  ┌─────────────┐  ← 堆疊底部（Stack Bottom）
            │  0x0000ABCD  │  ← 最早 PUSH 的資料
0x7FFF0018  ├─────────────┤
            │  0x1234BEEF  │
0x7FFF0010  ├─────────────┤
            │  0x0000000A  │
0x7FFF0008  ├─────────────┤
            │  0x00006789  │  ← 最後 PUSH 的資料（堆疊頂端）
            ├─────────────┤  ← RSP 指向這裡
            │   (未使用)   │
0x7FFF0000  └─────────────┘  ← 可用空間繼續往下延伸
```

### 6.1.2 PUSH 與 POP 的詳細行為

```asm
; PUSH src
; 等同於：
;   SUB RSP, 8
;   MOV [RSP], src

PUSH RAX        ; RSP -= 8，[RSP] = RAX

; POP dst
; 等同於：
;   MOV dst, [RSP]
;   ADD RSP, 8

POP RBX         ; RBX = [RSP]，RSP += 8
```

**堆疊操作步驟追蹤（以 PUSH 100, PUSH 200, POP 為例）：**

```
初始 RSP = 0x1000

PUSH 100:
  RSP = 0x1000 - 8 = 0x0FF8
  [0x0FF8] = 100
  堆疊：... | 100 |  ← RSP

PUSH 200:
  RSP = 0x0FF8 - 8 = 0x0FF0
  [0x0FF0] = 200
  堆疊：... | 200 | 100 |  ← RSP

POP EAX:
  EAX = [0x0FF0] = 200
  RSP = 0x0FF0 + 8 = 0x0FF8
  堆疊：... | 100 |  ← RSP
  （200 所在的記憶體內容仍存在，但不屬於堆疊了）
```

### 6.1.3 堆疊的多種用途

| 用途 | 說明 | 對應指令 |
|------|------|---------|
| 暫存資料 | 暫時儲存暫存器值 | `PUSH` / `POP` |
| 傳遞參數 | 超過 6 個的函式參數 | `PUSH` 後 `CALL` |
| 儲存返回位址 | `CALL` 時自動壓入 | `CALL` / `RET` |
| 區域變數 | 函式的區域變數空間 | `SUB RSP, n` |
| 保存旗標 | 保存與還原 RFLAGS | `PUSHFQ` / `POPFQ` |

## 6.2 副程式的呼叫與返回

### 6.2.1 CALL 指令的詳細行為

```asm
CALL label
; 等同於：
;   PUSH RIP      （將下一條指令的位址壓入堆疊）
;   JMP  label    （跳到 label）
```

```
呼叫前：RSP = 0x1000，RIP = 0x0040_1000

執行 CALL func（位於 0x0040_2000）：
  PUSH RIP → [0x0FF8] = 0x0040_1005（CALL 指令後的下一條指令位址）
  RSP = 0x0FF8
  RIP = 0x0040_2000（跳入函式）
```

### 6.2.2 RET 指令的詳細行為

```asm
RET
; 等同於：
;   POP RIP       （從堆疊彈出返回位址）
;   JMP RIP       （跳回呼叫者）
```

```
函式執行完畢，RSP = 0x0FF8，[0x0FF8] = 0x0040_1005

執行 RET：
  RIP = [0x0FF8] = 0x0040_1005
  RSP = 0x0FF8 + 8 = 0x1000
  跳回呼叫者繼續執行
```

### 6.2.3 CALL / RET 配對原則

```asm
; ✓ 正確：每個 CALL 對應一個 RET，堆疊保持平衡
my_func:
    PUSH RBP
    MOV  RBP, RSP
    ; ... 函式主體 ...
    POP  RBP
    RET              ; RSP 回到 CALL 前的狀態

; ✗ 錯誤：RET 前 RSP 指向錯誤位置
bad_func:
    PUSH RAX         ; 壓入一個值但沒有 POP
    RET              ; 會從堆疊讀到錯誤的返回位址！
```

**堆疊不平衡是最難除錯的錯誤之一。** 每個進入函式時的 PUSH，都必須在 RET 之前 POP。

## 6.3 堆疊框架（Stack Frame）

### 6.3.1 堆疊框架的概念

**堆疊框架（Stack Frame）** 是每個函式在堆疊上佔用的一塊區域，包含：

1. 返回位址（由 CALL 自動壓入）
2. 呼叫者的 RBP（由函式序言保存）
3. 區域變數空間
4. 可能的 callee-saved 暫存器

```
RSP 進入函式前（CALL 後）：

高位址
┌────────────────────┐
│    呼叫者的資料     │  ← 呼叫者的堆疊框架
├────────────────────┤
│    返回位址（RIP）  │  ← CALL 自動壓入  [RBP+8]
├────────────────────┤  ← RSP 此時指這裡
│                    │

執行函式序言後（PUSH RBP; MOV RBP, RSP; SUB RSP, n）：

高位址
┌────────────────────┐
│    呼叫者的資料     │
├────────────────────┤
│    返回位址（RIP）  │  [RBP+8]
├────────────────────┤
│    舊 RBP          │  [RBP+0]  ← RBP 指這裡
├────────────────────┤
│    區域變數 a      │  [RBP-4]
├────────────────────┤
│    區域變數 b      │  [RBP-8]
├────────────────────┤
│    區域變數 c      │  [RBP-12]
├────────────────────┤  ← RSP 指這裡
低位址
```

### 6.3.2 函式序言（Prologue）與結語（Epilogue）

```asm
; ── 函式序言（Prologue）──────────────────────────────
my_function:
    PUSH RBP            ; 保存呼叫者的 RBP
    MOV  RBP, RSP       ; 設定本函式的基底指標
    SUB  RSP, 32        ; 保留 32 bytes 給區域變數
                        ; （必須是 16 的倍數，確保堆疊對齊）

    ; 若需要使用 callee-saved 暫存器
    PUSH RBX
    PUSH R12

    ; ── 函式主體 ──────────────────────────────────
    ; 區域變數存取（以 RBP 為基準）：
    MOV DWORD [RBP-4],  10      ; 第一個 int 區域變數
    MOV DWORD [RBP-8],  20      ; 第二個 int 區域變數
    MOV QWORD [RBP-16], 0       ; 第一個 long 區域變數

    ; ── 函式結語（Epilogue）──────────────────────
    POP  R12            ; 還原 callee-saved（順序相反）
    POP  RBX
    MOV  RSP, RBP       ; 釋放區域變數空間
    POP  RBP            ; 還原呼叫者的 RBP
    RET
```

### 6.3.3 leave 指令

`LEAVE` 是函式結語的縮寫：

```asm
LEAVE
; 等同於：
;   MOV RSP, RBP   ; 釋放區域變數
;   POP RBP        ; 還原 RBP

; 因此結語可以簡化為：
LEAVE
RET
```

### 6.3.4 堆疊對齊（Stack Alignment）

**x86-64 System V ABI 要求：在 CALL 指令執行前，RSP 必須是 16 的倍數。**

```
CALL 指令執行 PUSH RIP（8 bytes）後，RSP 變成 16n - 8（非 16 倍數）
因此函式序言（PUSH RBP 再加 8）後 RSP 再次對齊到 16 倍數

若 SUB RSP, n 後對齊狀況：
  n = 8  → RSP - 8（在 PUSH RBP 後）已對齊，無需額外 padding
  n = 16 → ✓ 16 的倍數
  n = 24 → ✓（RSP = 16n - 8 - 8 - 24 = 16(n-2.5)...需計算）

安全做法：n 取 16 的倍數
```

```asm
; 若有 3 個 int 區域變數（3 × 4 = 12 bytes），應 SUB RSP, 16
; 而非 SUB RSP, 12，保持 16-byte 對齊

SUB RSP, 16     ; 而非 12，多 4 bytes 的 padding
```

## 6.4 參數傳遞方式

### 6.4.1 以暫存器傳遞（Linux System V AMD64 ABI）

整數與指標參數依序使用：**RDI、RSI、RDX、RCX、R8、R9**

```asm
; C 函式：long add3(long a, long b, long c)
; 呼叫 add3(10, 20, 30)

MOV RDI, 10         ; 第 1 個參數
MOV RSI, 20         ; 第 2 個參數
MOV RDX, 30         ; 第 3 個參數
CALL add3
; 返回值在 RAX

; 被呼叫函式：
add3:
    ; RDI = a = 10
    ; RSI = b = 20
    ; RDX = c = 30
    MOV RAX, RDI
    ADD RAX, RSI
    ADD RAX, RDX    ; RAX = 10 + 20 + 30 = 60
    RET
```

### 6.4.2 超過 6 個參數：使用堆疊

第 7 個以上的參數，**從右到左**依序壓入堆疊：

```c
// C 宣告
long func7(long a, long b, long c, long d,
           long e, long f, long g);
// 呼叫 func7(1,2,3,4,5,6,7)
```

```asm
; 第 7 個以上的參數先壓堆疊（由右到左）
PUSH 7              ; 第 7 個參數 g（最後壓）

; 前 6 個用暫存器
MOV RDI, 1         ; a
MOV RSI, 2         ; b
MOV RDX, 3         ; c
MOV RCX, 4         ; d
MOV R8,  5         ; e
MOV R9,  6         ; f
CALL func7

; CALL 結束後清理堆疊（呼叫者負責）
ADD RSP, 8         ; 清除一個 QWORD 的堆疊參數

; ── 在 func7 內部存取堆疊參數 ──
func7:
    PUSH RBP
    MOV  RBP, RSP
    ; RDI=a, RSI=b, RDX=c, RCX=d, R8=e, R9=f
    ; [RBP+16] = g（CALL 壓了返回位址，PUSH RBP 又壓了 RBP）
    MOV RAX, [RBP+16]  ; 取得第 7 個參數 g
    ; ...
    POP RBP
    RET
```

**堆疊上的參數佈局：**

```
呼叫 func7 後：
  [RBP+16] = g（第 7 個參數）
  [RBP+8]  = 返回位址
  [RBP+0]  = 舊 RBP
```

### 6.4.3 以值傳遞 vs 以指標傳遞

```asm
; ── 以值傳遞（Pass by Value）──
; C：void double_val(int x)  →  修改不影響呼叫者
MOV EDI, [my_var]      ; 傳入 my_var 的值的副本
CALL double_val

; ── 以指標傳遞（Pass by Pointer / Reference）──
; C：void double_ref(int *x)  →  修改會影響呼叫者
LEA RDI, [my_var]      ; 傳入 my_var 的位址
CALL double_ref
; double_ref 內部：MOV EAX,[RDI] → ADD EAX,EAX → MOV [RDI],EAX
```

### 6.4.4 Windows x64 呼叫慣例（對比參考）

Windows 使用不同的 ABI：

| 項目 | Linux System V | Windows x64 |
|------|---------------|-------------|
| 參數暫存器 | RDI, RSI, RDX, RCX, R8, R9 | RCX, RDX, R8, R9 |
| 回傳值 | RAX | RAX |
| Shadow Space | 無 | 呼叫者需在 RSP 預留 32 bytes |
| Callee-saved | RBX, RBP, R12-R15 | RBX, RBP, RDI, RSI, R12-R15 |

### 6.5.1 整數與指標回傳

## 6.6 遞迴函式

### 6.6.1 遞迴的本質

每一次遞迴呼叫都會在堆疊上建立一個新的堆疊框架，直到遞迴終止條件成立後，框架依序被釋放。

### 6.6.2 範例：遞迴計算階乘

```c
// C 程式碼
long factorial(long n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```

```asm
; 組合語言實作
; 參數：RDI = n
; 回傳：RAX = n!

factorial:
    ; 序言
    PUSH RBP
    MOV  RBP, RSP
    PUSH RBX                ; callee-saved，用來存 n

    ; if (n <= 1) return 1
    CMP  RDI, 1
    JG   .recurse           ; n > 1 則遞迴
    MOV  RAX, 1             ; 基底情況：return 1
    JMP  .done

.recurse:
    MOV  RBX, RDI           ; RBX = n（保存，因為後面會改 RDI）
    DEC  RDI                ; 參數 = n - 1
    CALL factorial          ; RAX = factorial(n - 1)
    IMUL RAX, RBX           ; RAX = n * factorial(n-1)

.done:
    POP  RBX                ; 還原 callee-saved
    POP  RBP
    RET
```

### 6.6.3 遞迴的堆疊展開過程（以 factorial(4) 為例）

```
呼叫 factorial(4)：
  堆疊框架 4：n=4，RBX=4，返回位址=...
    呼叫 factorial(3)：
      堆疊框架 3：n=3，RBX=3，返回位址=.recurse+5
        呼叫 factorial(2)：
          堆疊框架 2：n=2，RBX=2，返回位址=.recurse+5
            呼叫 factorial(1)：
              堆疊框架 1：n=1（基底情況）
              返回 RAX = 1
          ← RAX=1，RBX=2，執行 IMUL RAX,RBX → RAX=2
          返回 RAX = 2
        ← RAX=2，RBX=3，執行 IMUL RAX,RBX → RAX=6
        返回 RAX = 6
      ← RAX=6，RBX=4，執行 IMUL RAX,RBX → RAX=24
      返回 RAX = 24

最終結果：factorial(4) = 24
```

### 6.6.4 堆疊溢位（Stack Overflow）

遞迴深度過深時，堆疊空間耗盡：

```
Linux 預設堆疊大小：8 MB（可用 ulimit -s 查詢）
每個框架約佔 16～64 bytes
→ 最大遞迴深度：約 100,000 ～ 500,000 層

factorial(1,000,000)  →  Stack Overflow！
```

解決方案：
1. **尾遞迴最佳化（Tail Call Optimization）**：若最後一步只是 `return func()`，編譯器可將其轉換為跳躍而不是呼叫
2. **改為迭代版本**：用迴圈取代遞迴

## 6.7 尾呼叫最佳化（Tail Call Optimization）

### 6.7.1 什麼是尾呼叫？

若函式的最後一個動作是呼叫另一個函式（且不對結果做任何處理），這個呼叫就是**尾呼叫（Tail Call）**。

```c
// 非尾遞迴：最後還需要做乘法
long factorial(long n) {
    return n * factorial(n - 1);  // 不是尾呼叫（還需 × n）
}

// 尾遞迴版本（需要累加器）
long factorial_tail(long n, long acc) {
    if (n <= 1) return acc;
    return factorial_tail(n - 1, acc * n);  // 是尾呼叫
}
```

### 6.7.2 尾呼叫的組語最佳化

```asm
; 一般遞迴：每層都需要 CALL（建立新框架）
; factorial_tail 的尾呼叫版本可轉換為跳躍：

factorial_tail:
    ; 參數：RDI = n，RSI = acc
    CMP  RDI, 1
    JLE  .base_case

    ; 尾呼叫最佳化：不用 CALL，直接修改參數後 JMP
    IMUL RSI, RDI           ; acc = acc * n
    DEC  RDI                ; n = n - 1
    JMP  factorial_tail     ; 重用目前框架！（不建立新框架）

.base_case:
    MOV RAX, RSI            ; return acc
    RET
```

**尾呼叫最佳化的好處：** 遞迴深度再深也不會溢位，因為每次都重用同一個堆疊框架。

## 6.8 內嵌函式與行內組語

### 6.8.1 內嵌函式呼叫（Inline Function）

編譯器有時會將短小函式展開為行內程式碼，省去 CALL/RET 的開銷：

```c
// C 的 static inline 函式
static inline int square(int x) { return x * x; }

int main() {
    int y = square(5);   // 編譯器可能展開為：EAX = 5; IMUL EAX, EAX
}
```

```asm
; 編譯器展開後（不產生 CALL）：
MOV EAX, 5
IMUL EAX, EAX   ; EAX = 25，省去一次函式呼叫的開銷
```

### 6.8.2 與 C 語言互動（extern "C"）

```c
// C 語言呼叫組語函式
extern long my_asm_func(long a, long b);  // 宣告

long result = my_asm_func(10, 20);
```

```asm
; 組語端（NASM）
global my_asm_func

my_asm_func:
    ; RDI = a = 10，RSI = b = 20
    MOV RAX, RDI
    ADD RAX, RSI        ; RAX = a + b
    RET
```

## 6.9 變長引數（Variadic Functions）

### 6.9.1 printf 的呼叫慣例

`printf` 等變長引數函式需要特殊處理：

```asm
; 呼叫 printf("Value: %d\n", 42)
section .data
    fmt DB "Value: %d", 0x0a, 0    ; 格式字串

section .text
    LEA RDI, [fmt]      ; 第 1 參數：格式字串
    MOV RSI, 42         ; 第 2 參數：整數值
    XOR EAX, EAX        ; ★ 重要：EAX = 浮點參數個數（無 XMM 參數則為 0）
    CALL printf

; 若有浮點參數：
    MOV EAX, 1          ; 有 1 個 XMM 參數
    MOVSD XMM0, [pi]    ; 浮點值放在 XMM0
    CALL printf
```

> 在 System V ABI 中，呼叫變長引數函式前，`EAX` 必須設定為「使用的 XMM 暫存器數量」。忘記清零 `EAX` 是組語呼叫 `printf` 時的常見錯誤。

## 6.10 堆疊的安全與保護

### 6.10.1 緩衝區溢位（Buffer Overflow）

```c
// 脆弱的 C 程式碼
void vulnerable(char *input) {
    char buf[16];
    strcpy(buf, input);   // 若 input > 16 bytes，會覆蓋堆疊！
}
```

```
攻擊者輸入 "AAAAAAAAAAAAAAAAAAAA\x42\x20\x40\x00"（過長字串）：

堆疊框架：
  [RBP-16] buf[0..15]  ← 被填滿後繼續溢出 ↓
  [RBP-0]  舊 RBP      ← 被覆蓋為 'AAAA...'
  [RBP+8]  返回位址    ← 被覆蓋為攻擊者控制的位址！
```

### 6.10.2 現代保護機制

| 機制 | 說明 |
|------|------|
| **Stack Canary（金絲雀）** | 在返回位址前放置隨機值，返回時驗證是否被修改 |
| **ASLR** | 隨機化堆疊基底位址，使攻擊者難以預測位址 |
| **NX Bit（DEP）** | 標記堆疊為不可執行，防止 shellcode 執行 |
| **Shadow Stack（CET）** | Intel 的控制流程強制技術，維護一份影子返回位址堆疊 |

```asm
; GCC 的 Stack Canary 實作（簡化版）：
my_func:
    PUSH RBP
    MOV  RBP, RSP
    SUB  RSP, 32

    ; 讀取 canary 值（從 FS:[0x28] 取得隨機值）
    MOV  RAX, QWORD [FS:0x28]
    MOV  [RBP-8], RAX           ; 存在 buf 和返回位址之間

    ; ... 函式主體 ...

    ; 返回前驗證 canary
    MOV  RAX, [RBP-8]
    XOR  RAX, QWORD [FS:0x28]  ; 若 canary 被修改，XOR 結果不為 0
    JNZ  __stack_chk_fail       ; 觸發安全錯誤

    MOV  RSP, RBP
    POP  RBP
    RET
```

## 6.11 常見堆疊操作慣用法

### 6.11.1 保存與還原多個暫存器

```asm
; 進入函式：保存所有需要用到的 callee-saved 暫存器
func:
    PUSH RBP
    PUSH RBX
    PUSH R12
    PUSH R13
    PUSH R14
    PUSH R15
    ; 注意：PUSH 了 6 個 QWORD（48 bytes），加上返回位址（8 bytes）= 56 bytes
    ; RSP 與 CALL 前相差 56 bytes（不是 16 的倍數）
    ; 若需要呼叫其他函式，可能需要 SUB RSP, 8 對齊

    ; ... 函式主體（可自由使用 RBX, R12-R15）...

    ; 還原（順序必須相反）
    POP R15
    POP R14
    POP R13
    POP R12
    POP RBX
    POP RBP
    RET
```

### 6.11.2 在堆疊上建立臨時陣列

```asm
; 在堆疊上建立 int arr[10]（40 bytes）
    SUB RSP, 48         ; 40 bytes 資料 + 8 bytes padding（對齊）
    ; RSP 指向 arr 的開始
    LEA RDI, [RSP]      ; RDI = arr 的位址

    ; 使用 arr：[RSP + i*4] 為 arr[i]
    MOV DWORD [RSP+0],  1
    MOV DWORD [RSP+4],  2
    MOV DWORD [RSP+8],  3

    ; 用完後釋放
    ADD RSP, 48
```

### 6.11.3 用堆疊傳遞結構體

```asm
; 在堆疊上建立 struct Point { int x; int y; }，傳給函式
    SUB RSP, 16                 ; 保留結構體空間（對齊到 16）
    MOV DWORD [RSP+0], 10       ; x = 10
    MOV DWORD [RSP+4], 20       ; y = 20
    MOV RDI, RSP                ; 傳入指標
    CALL process_point
    ADD RSP, 16                 ; 釋放結構體空間
```

## 6.12 本章小結

**堆疊特性：**
- 向低位址成長，RSP 始終指向堆疊頂端
- PUSH：RSP -= 8，寫入；POP：讀取，RSP += 8
- 每次 CALL 自動壓入返回位址，每次 RET 自動彈出

**堆疊框架：**
- 由序言（PUSH RBP; MOV RBP, RSP; SUB RSP, n）建立
- RBP 固定作為基準，區域變數在 [RBP-n]，堆疊參數在 [RBP+16] 以上
- 結語（MOV RSP, RBP; POP RBP; RET）或 LEAVE + RET 釋放

**參數傳遞（Linux System V AMD64）：**
- 前 6 個整數參數：RDI → RSI → RDX → RCX → R8 → R9
- 第 7 個以上：由右到左壓堆疊，呼叫後由呼叫者清理（ADD RSP, n）
- 回傳值：RAX（128-bit 時加 RDX）

**遞迴：**
- 每層遞迴建立一個新堆疊框架，遞迴深度受限於堆疊大小
- 尾遞迴可被最佳化為 JMP（不建立新框架），有效避免堆疊溢位
- 尾呼叫的必要條件：函式的最後一個動作是 return func(...)，且不對結果做任何運算

**安全：**
- Stack Canary 偵測堆疊溢位攻擊
- ASLR 隨機化位址、NX Bit 防止堆疊上的程式碼執行
