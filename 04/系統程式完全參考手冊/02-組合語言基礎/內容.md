# 第 2 章　組合語言基礎

> **本章目標：** 深入理解組合語言的程式結構、語法規則、資料定義、數值表示、虛擬指令、記憶體佈局（小端序），以及兩遍組譯器的完整運作流程。

---

## 2.1 組合語言的定位與特性

### 2.1.1 三層語言的關係

程式語言依據「與硬體的距離」可分為三個層次，組合語言正好處於中間：

```
高階語言（C / Java / Python）
    ↕  編譯器翻譯
組合語言（Assembly Language）
    ↕  組譯器翻譯（幾乎一對一）
機器語言（Machine Language / Binary）
    ↕  CPU 直接執行
```

同一件事在三個層次的表達方式：

| 層次 | 程式碼 | 說明 |
|------|--------|------|
| 高階語言 | `x = a + b;` | 人類可讀，與硬體無關 |
| 組合語言 | `MOV EAX, [a]` → `ADD EAX, [b]` → `MOV [x], EAX` | 符號化，與架構相關 |
| 機器語言 | `8B 05 04 20 00 00 03 05 08 20 00 00 89 05 0C 20 00 00` | CPU 直接理解的二進位 |

### 2.1.2 為什麼要學組合語言？

雖然現代大多數軟體都以高階語言撰寫，但組合語言在以下場景不可或缺：

1. **作業系統核心**：開機程式（Bootloader）、中斷處理常式、部分排程器程式碼，必須直接控制硬體。
2. **嵌入式系統**：資源受限的微控制器（如 8051、ARM Cortex-M0）上，記憶體與效能要求極嚴苛。
3. **驅動程式**：與特定硬體暫存器直接互動，高階語言抽象層會帶來不必要的開銷。
4. **逆向工程 / 資安**：分析惡意程式、找漏洞（如 Buffer Overflow）需要讀懂反組譯輸出。
5. **理解編譯器輸出**：使用 `gcc -S` 觀察編譯器如何最佳化程式碼，加深對語言底層的理解。
6. **SIMD 最佳化**：手動撰寫 SSE/AVX 向量指令，達到自動向量化無法實現的效能。

### 2.1.3 組合語言的特性

| 特性 | 說明 |
|------|------|
| **與架構綁定** | x86 的組語無法在 ARM 上執行，每種 ISA 有自己的助記符 |
| **一對一對應** | 通常每條組語指令對應一條機器指令（虛擬指令除外） |
| **完全控制** | 可精確控制每個暫存器、每個位元組的記憶體操作 |
| **無型別系統** | 沒有 int / float 等型別概念，程式設計師自行管理資料格式 |
| **難以維護** | 程式碼冗長、可讀性低，大型專案幾乎不以純組語撰寫 |

## 2.2 組語程式的語法結構

### 2.2.1 四欄位格式

每一行組合語言最多由四個欄位組成，**欄位之間以空白（空格或 Tab）分隔**：

```
[標號:]   助記符   [運算元1 [, 運算元2]]   [; 註解]
```

完整標示各欄位的範例：

```
┌────────┐ ┌──────┐ ┌────────────┐ ┌─────────────────────┐
│  標號  │ │助記符│ │  運算元    │ │       註解          │
└────────┘ └──────┘ └────────────┘ └─────────────────────┘
  START:     MOV     EAX, 10         ; 將立即值 10 存入 EAX
  LOOP:      ADD     EAX, EBX        ; EAX = EAX + EBX
             JNZ     LOOP            ; 若非零則跳到 LOOP
             RET                     ; 無運算元的指令
```

### 2.2.2 各欄位詳細說明

#### 標號（Label）

標號是某個記憶體位址的**符號名稱**，讓程式設計師不需記住實際位址。

```asm
func_start:          ; 標記函式進入點的位址
    mov eax, 0

loop_top:            ; 標記迴圈開始的位址
    add eax, 1
    cmp eax, 10
    jl  loop_top    ; 跳回 loop_top 所在的位址

data_section:        ; 資料標號（在 .data 區段）
    db "hello"
```

標號的規則：
- 以字母或底線 `_` 開頭
- 可包含字母、數字、底線、句點 `.`
- **大小寫敏感**（`Start` 和 `start` 是不同的標號）
- 以冒號 `:` 結尾（某些格式允許省略）
- 不可與助記符或關鍵字同名

#### 助記符（Mnemonic）

助記符是 CPU 指令的**符號縮寫**，組譯器將其翻譯為對應的機器碼 opcode。

```asm
MOV   ; Move（資料傳送）
ADD   ; Add（加法）
SUB   ; Subtract（減法）
CMP   ; Compare（比較，設定旗標）
JMP   ; Jump（無條件跳躍）
CALL  ; Call subroutine（呼叫副程式）
RET   ; Return（返回）
```

助記符也可能是**虛擬指令（Pseudo-instruction / Directive）**，這類指令不產生機器碼，而是給組譯器本身的指示（詳見 2.4 節）：

```asm
DB    ; Define Byte（定義資料）
EQU   ; Equate（定義常數）
SECTION ; 切換區段
GLOBAL  ; 宣告全域符號
```

#### 運算元（Operand）

運算元指定指令的**操作對象**，可以是：

| 類型 | 範例 | 說明 |
|------|------|------|
| 立即值 | `MOV EAX, 42` | 數值常數，直接編碼在指令中 |
| 暫存器 | `MOV EAX, EBX` | CPU 內部暫存器 |
| 記憶體位址 | `MOV EAX, [var]` | 方括號 `[]` 表示「存取該位址的內容」 |
| 標號 | `JMP loop_top` | 跳躍目標位址，由組譯器計算 |

**運算元的順序（Intel 語法 vs AT&T 語法）：**

```asm
; Intel 語法（NASM / MASM 使用）：目的 ← 來源
MOV  EAX, EBX      ; EAX = EBX（左邊是目的，右邊是來源）
ADD  EAX, 5        ; EAX = EAX + 5

; AT&T 語法（GAS / GCC 內嵌組語使用）：來源 → 目的
movl %ebx, %eax    ; EAX = EBX（左邊是來源，右邊是目的）
addl $5, %eax      ; EAX = EAX + 5（立即值加 $，暫存器加 %）
```

> ⚠️ **重要：** 本手冊使用 Intel 語法（NASM 格式）。若閱讀 GCC 輸出（`gcc -S`），需注意它預設使用 AT&T 語法，運算元順序相反。

#### 註解（Comment）

```asm
; 這是單行註解（NASM / MASM）
# 這也是單行註解（GAS）

MOV EAX, 0  ; 行尾註解：清除 EAX
```

良好的註解應說明**為什麼**，而不只是**做什麼**：

```asm
; 不好的註解（只重複了程式碼本身）
MOV ECX, 10     ; 將 10 存入 ECX

; 好的註解（說明用途與意義）
MOV ECX, 10     ; 迴圈計數器，執行 10 次
```

## 2.3 程式區段（Section）

### 2.3.1 為什麼需要區段？

區段（Section / Segment）的設計目的是將不同性質的內容分開存放，讓作業系統能對它們設定不同的記憶體保護屬性：

```
.text  區段  → 唯讀 + 可執行（防止程式碼被意外修改）
.data  區段  → 可讀寫 + 不可執行（資料不應被當作指令執行）
.bss   區段  → 可讀寫 + 不可執行（與 .data 類似，但不佔磁碟空間）
```

### 2.3.2 三大區段詳解

#### .text — 程式碼區段

```asm
section .text
    global _start       ; 宣告 _start 為全域可見（連結器需要）

_start:
    ; 所有可執行的指令放在這裡
    mov eax, 1
    syscall
```

- 存放所有 CPU 可執行的機器指令
- 在磁碟（ELF 檔）和記憶體中都佔有空間
- 作業系統設定此區段為**唯讀**，防止自我修改的程式碼（除非特別設定）
- 多個 process 可以共享同一份 `.text`（shared library 的基礎）

#### .data — 已初始化資料區段

```asm
section .data
    greeting    db  "Hello, World!", 0x0a   ; 有初始值的字串
    max_count   dd  100                     ; 有初始值的整數
    pi_approx   dd  3                       ; 近似值（整數）
    enabled     db  1                       ; 旗標：1 = true
```

- 存放**有初始值**的全域變數和常數
- 初始值會直接儲存在 ELF 檔案中（磁碟上佔空間）
- 執行時映射到記憶體，可以被程式讀寫

#### .bss — 未初始化資料區段

```asm
section .bss
    input_buffer    resb    256     ; 輸入緩衝區，256 bytes
    result_array    resd    100     ; 100 個整數的陣列
    error_flag      resb    1       ; 1 byte 的旗標
```

- 存放**沒有初始值**的全域變數
- **在 ELF 磁碟檔中只記錄大小，不記錄實際內容**（節省磁碟空間）
- 作業系統在載入程式時，自動將這塊記憶體清為全零
- `.bss` 全名：**Block Started by Symbol**（源自 1950 年代組語慣例）

**為什麼 .bss 能節省空間？**

```
假設有一個全域陣列 int arr[10000]（未初始化）：

.data 做法：ELF 檔需要儲存 40,000 bytes 的零值資料
.bss  做法：ELF 檔只記錄「這裡需要 40,000 bytes 的空間」
            → ELF 檔小得多，作業系統載入時才配置並清零
```

### 2.3.3 區段比較總表

| 屬性 | `.text` | `.data` | `.bss` |
|------|---------|---------|--------|
| 內容 | 機器指令 | 已初始化資料 | 未初始化資料 |
| ELF 檔佔用 | 是 | 是 | 只記錄大小 |
| 記憶體佔用 | 是 | 是 | 是（執行時配置）|
| 初始值 | — | 由程式設計師指定 | 全 0（OS 保證）|
| 執行時可寫 | 否（預設）| 是 | 是 |
| 可執行 | 是 | 否 | 否 |

### 2.3.4 記憶體中的程式佈局

程式載入到記憶體後，各區段的相對位置如下（Linux x86-64）：

```
高位址
┌─────────────────────┐  ← 0xFFFFFFFFFFFFFFFF
│    Kernel Space     │    (使用者程式不可存取)
├─────────────────────┤  ← 0x7FFFFFFFFFFF（近似）
│       Stack         │    區域變數、函式呼叫資訊
│         ↓           │    向低位址成長
│   (未使用空間)        │
│         ↑           │
│        Heap         │    動態配置記憶體（malloc）
│                     │    向高位址成長
├─────────────────────┤
│        .bss         │    全域未初始化變數（清為 0）
├─────────────────────┤
│       .data         │    全域已初始化變數
├─────────────────────┤
│       .text         │    程式碼（唯讀+可執行）
└─────────────────────┘  ← 0x0000000000400000（近似）
低位址
```

## 2.4 資料定義虛擬指令

### 2.4.1 虛擬指令 vs 真實指令

**真實指令（Real Instruction）：** 翻譯為機器碼，由 CPU 執行。
```asm
MOV EAX, 5    → 機器碼：B8 05 00 00 00
ADD EAX, EBX  → 機器碼：01 D8
```

**虛擬指令（Pseudo-instruction / Directive）：** 給組譯器的指示，不產生機器碼。
```asm
DB 65         → 在記憶體中放一個 byte 值 65（字元 'A'）
EQU 100       → 定義一個符號常數，不佔記憶體
SECTION .text → 告訴組譯器切換到 .text 區段
```

### 2.4.2 資料定義指令（Define）

#### DB — Define Byte（1 byte）

```asm
; 單一數值
a   DB  0           ; 值為 0
b   DB  255         ; 值為 0xFF
c   DB  -1          ; 有號數 -1，二補數表示為 0xFF
d   DB  0x41        ; 十六進位，值為 65（'A'）

; 字元
ch  DB  'A'         ; 等同於 DB 65

; 字串（字元陣列）
msg DB  "Hello"     ; 5 個連續 bytes：72 65 6C 6C 6F
str DB  "Hi", 0     ; C 風格字串：以 null byte(0) 結尾
nl  DB  "Hi", 0x0a  ; 字串 + 換行字元

; 多個值（陣列）
arr DB  1, 2, 3, 4, 5   ; 5 個 bytes
mix DB  'A', 0x0a, 0     ; 混合字元與數值
```

**記憶體佈局（字串 "Hi" + null）：**
```
位址  0x00  0x01  0x02
值    0x48  0x69  0x00
字元  'H'   'i'   NUL
```

#### DW — Define Word（2 bytes，小端序）

```asm
w1  DW  0x1234      ; 在記憶體中：34 12（小端序！）
w2  DW  1000        ; 0x03E8 → 記憶體：E8 03
w3  DW  -1          ; 0xFFFF → 記憶體：FF FF
arr DW  10, 20, 30  ; 6 bytes：0A 00  14 00  1E 00
```

**小端序（Little-Endian）說明：**

x86 架構使用小端序，意思是一個多位元組的數值，**低位 byte 存在低位址**：

```
數值 0x1234：
  高位 byte：0x12
  低位 byte：0x34

小端序儲存在位址 0x0100：
  位址 0x0100 → 0x34  （低位 byte 在前）
  位址 0x0101 → 0x12  （高位 byte 在後）

讀取時從高到低：12 34 → 0x1234 ✓
```

#### DD — Define Double Word（4 bytes）

```asm
d1  DD  0x12345678  ; 記憶體：78 56 34 12（小端序）
d2  DD  100000      ; 4 bytes 整數
d3  DD  3.14        ; 單精度浮點數（IEEE 754）
arr DD  0, 0, 0     ; 清零的 3 個 DWORD
```

#### DQ — Define Quad Word（8 bytes）

```asm
q1  DQ  0x123456789ABCDEF0  ; 8 bytes，64 位元整數
q2  DQ  3.14159265358979    ; 雙精度浮點數（IEEE 754）
```

#### DT — Define Ten Bytes（10 bytes）

```asm
t1  DT  3.14159265358979    ; 擴展精度浮點數（x87 FPU）
```

### 2.4.3 空間保留指令（Reserve）

Reserve 指令用於 `.bss` 區段，只分配空間，不設定初始值（OS 保證清零）：

| 指令 | 每單位大小 | 範例 | 保留空間 |
|------|-----------|------|---------|
| `RESB n` | 1 byte | `buf RESB 256` | 256 bytes |
| `RESW n` | 2 bytes | `arr RESW 50` | 100 bytes |
| `RESD n` | 4 bytes | `int_arr RESD 10` | 40 bytes |
| `RESQ n` | 8 bytes | `long_arr RESQ 8` | 64 bytes |

```asm
section .bss
    ; 輸入緩衝區
    stdin_buf   RESB    1024    ; 1 KB 輸入緩衝
    
    ; 整數陣列
    scores      RESD    100     ; 100 個 32-bit 整數 = 400 bytes
    
    ; 查找表
    lookup      RESB    256     ; 256 個 byte 值的查找表
    
    ; 結構體佔位（手動模擬 C struct）
    ; struct Point { int x; int y; int z; }  共 12 bytes
    point_x     RESD    1       ; int x
    point_y     RESD    1       ; int y
    point_z     RESD    1       ; int z
```

### 2.4.4 EQU — 常數定義

`EQU` 定義一個符號常數，**不佔用任何記憶體空間**，純粹是組譯時期的名稱替換：

```asm
; 基本用法
MAX_SIZE    EQU 256
NEWLINE     EQU 0x0a
NULL        EQU 0
TRUE        EQU 1
FALSE       EQU 0

; 使用已定義的常數
buffer      RESB MAX_SIZE       ; 等同於 RESB 256
```

**`$` 符號 — 目前位置計數器（Current Location Counter）：**

`$` 在 NASM 中代表**當前指令或資料的位址**，常用於計算長度：

```asm
section .data
    msg     DB  "Hello, World!", 0x0a
    msg_len EQU $ - msg       
    ; $ 指向 msg 定義結束後的位置
    ; $ - msg = (msg 的結束位址) - (msg 的起始位址) = 14

    arr     DD  10, 20, 30, 40, 50
    arr_size    EQU $ - arr           ; = 5 * 4 = 20 bytes
    arr_count   EQU ($ - arr) / 4    ; = 20 / 4 = 5 個元素
```

**EQU vs DB 的差異：**

```asm
MAX     EQU 100     ; 純符號，不佔記憶體，只在組譯時期存在
limit   DB  100     ; 在記憶體中配置 1 byte，值為 100

; 使用方式的差異：
MOV EAX, MAX        ; EAX = 100（立即值）
MOV EAX, [limit]    ; EAX = 記憶體 limit 位址的內容
```

## 2.5 數值表示方式

組譯器支援多種數值格式，方便在不同情境下直接使用適合的表示法。

### 2.5.1 整數格式

```asm
; ── 十進位（預設，無前綴）
MOV EAX, 255
MOV EBX, 1000000

; ── 十六進位
MOV EAX, 0xFF          ; NASM：0x 前綴
MOV EAX, 0FFh          ; MASM：h 後綴（必須以數字開頭！）
MOV EAX, $FF           ; 某些舊式組譯器

; ── 二進位
MOV AL,  0b11001010    ; NASM：0b 前綴
MOV AL,  11001010b     ; MASM：b 後綴

; ── 八進位
MOV EAX, 0o377         ; NASM：0o 前綴（255 的八進位）
MOV EAX, 377q          ; MASM：q 後綴

; ── 字元（ASCII 碼）
MOV AL,  'A'           ; AL = 65（0x41）
MOV AL,  'a'           ; AL = 97（0x61）
MOV AX,  'Hi'          ; AX = 0x6948（小端序：'H'=0x48, 'i'=0x69）
```

### 2.5.2 常用 ASCII 值速查

| 字元 | 十進位 | 十六進位 | 說明 |
|------|--------|---------|------|
| `NUL` | 0 | 0x00 | 字串結尾（null terminator） |
| `\t` | 9 | 0x09 | 水平 Tab |
| `\n` | 10 | 0x0A | 換行（Line Feed） |
| `\r` | 13 | 0x0D | 歸位（Carriage Return） |
| `' '` | 32 | 0x20 | 空格 |
| `'0'` | 48 | 0x30 | 數字 0 |
| `'9'` | 57 | 0x39 | 數字 9 |
| `'A'` | 65 | 0x41 | 大寫 A |
| `'Z'` | 90 | 0x5A | 大寫 Z |
| `'a'` | 97 | 0x61 | 小寫 a |
| `'z'` | 122 | 0x7A | 小寫 z |

**快速換算技巧：**
- 大寫 → 小寫：`OR AL, 0x20`（差值恰好是 32 = 0x20）
- 小寫 → 大寫：`AND AL, 0xDF`（清除第 5 位元）
- 數字字元 → 數值：`SUB AL, '0'`（'7' - '0' = 7）

### 2.5.3 有號數 vs 無號數

組合語言本身**不區分有號數和無號數**，相同的位元模式依解讀方式而有不同意義：

```asm
; 存入 0xFF
MOV AL, 0xFF   ; 位元模式：1111 1111

; 解讀一：無號數（UNSIGNED）
; AL = 255

; 解讀二：有號數（SIGNED，二補數）
; AL = -1
```

**二補數（Two's Complement）快速轉換：**

```
正數 → 負數：取反（NOT）後加 1
  127 = 0111 1111
  取反 = 1000 0000
  加一 = 1000 0001 = -127

OR 更快方法：
  -N = ~N + 1
  -1 = 0xFF = 1111 1111（全 1）
  -2 = 0xFE = 1111 1110
  -128 = 0x80 = 1000 0000（8-bit 最小值）
```

**各資料寬度的數值範圍：**

| 大小 | 無號數範圍 | 有號數範圍 |
|------|-----------|-----------|
| 8-bit（byte）| 0 ～ 255 | -128 ～ 127 |
| 16-bit（word）| 0 ～ 65,535 | -32,768 ～ 32,767 |
| 32-bit（dword）| 0 ～ 4,294,967,295 | -2,147,483,648 ～ 2,147,483,647 |
| 64-bit（qword）| 0 ～ 2^64-1 | -2^63 ～ 2^63-1 |

## 2.6 標號（Label）的使用與作用域

### 2.6.1 標號的本質

標號是**位址的別名**，讓程式設計師以有意義的名稱代替難以記憶的數字位址：

```asm
; 沒有標號的跳躍（不實際，只是示意）
JMP 0x0010    ; 跳到位址 0x10，這個數字是什麼意思？

; 有標號的跳躍（清楚易懂）
JMP main_loop ; 跳到 main_loop，語意明確
```

### 2.6.2 標號的類型

#### 全域標號（Global Label）

不加前綴，整個程式可見：

```asm
global _start       ; 宣告對外部（連結器）可見

_start:             ; 程式進入點
    call main_func

main_func:          ; 全域標號，同一檔案中可以直接使用
    mov eax, 0
    ret
```

#### 本地標號（Local Label）

NASM 中以 `.` 開頭，**作用域限制在上一個全域標號內**，避免命名衝突：

```asm
func_a:             ; 全域標號
    mov ecx, 10
.loop:              ; 本地標號：完整名稱為 func_a.loop
    dec ecx
    jnz .loop       ; 跳回 func_a.loop
    ret

func_b:             ; 新的全域標號，作用域切換
.loop:              ; 本地標號：完整名稱為 func_b.loop
    ; 這裡的 .loop 與 func_a.loop 不衝突！
    nop
    ret
```

#### 數字標號（Numeric Label，NASM）

臨時使用，可以重複定義，以 `1b`（backward）和 `1f`（forward）引用：

```asm
1:                  ; 標號 1
    dec ecx
    jnz 1b          ; 跳回最近的「1:」標號（b = backward）

    jmp 2f          ; 跳到最近的「2:」標號（f = forward）

2:                  ; 標號 2（前向參考）
    nop
```

### 2.6.3 標號的進階使用

**標號作為資料位址：**

```asm
section .data
    array   DD  10, 20, 30, 40, 50

section .text
    ; 取得陣列第三個元素（index 2）
    ; 每個 DD = 4 bytes，所以偏移 = 2 * 4 = 8
    mov eax, [array + 8]    ; EAX = 30

    ; 或用索引暫存器（更靈活）
    mov ebx, 2              ; index = 2
    mov eax, [array + ebx*4]; EAX = array[2] = 30
```

**前向參考（Forward Reference）：**

```asm
    jmp skip_data           ; 跳過資料區（skip_data 尚未定義）

    ; 把常數嵌在 .text 區段中（舊式技巧）
magic_number:
    DD  0xDEADBEEF

skip_data:                  ; 這裡才定義 skip_data
    ; 程式從這裡繼續執行
```

前向參考是兩遍組譯存在的主要原因（詳見 2.8 節）。

## 2.7 常用控制類虛擬指令

### 2.7.1 區段控制

```asm
section .data           ; NASM 語法
segment .data           ; 也可使用 segment（同義）

[section .text]         ; 方括號語法（較舊）
```

### 2.7.2 符號可見性

```asm
global _start           ; _start 對連結器可見（進入點）
global func_a, func_b   ; 多個符號一起宣告

extern printf           ; 宣告 printf 是外部定義的符號
extern malloc, free     ; 同時宣告多個外部符號
```

### 2.7.3 位元模式設定

```asm
BITS 16     ; 產生 16-bit 程式碼（DOS、Bootloader）
BITS 32     ; 產生 32-bit 程式碼（x86 Protected Mode）
BITS 64     ; 產生 64-bit 程式碼（x86-64，現代系統）
```

### 2.7.4 起始位址設定

```asm
ORG 0x7C00  ; 設定起始位址為 0x7C00（BIOS 載入 Bootloader 的位址）
ORG 0x100   ; COM 格式程式的慣例起始位址
```

`ORG` 告訴組譯器：「假設這段程式碼會被載入到此位址」，影響所有位址計算。

### 2.7.5 條件組譯（Conditional Assembly）

```asm
%define DEBUG_MODE      ; 定義一個符號（類似 C 的 #define）

%ifdef DEBUG_MODE
    ; 只有在 DEBUG_MODE 定義時才組譯這段
    mov eax, 1
    syscall             ; 印出除錯資訊
%else
    ; 否則組譯這段
    nop
%endif

%define BUFFER_SIZE 1024
buf RESB BUFFER_SIZE    ; 等同於 RESB 1024
```

## 2.8 完整程式範例：Hello World 逐行解析

### 2.8.1 Linux x86-64（NASM）完整版

```asm
; ─────────────────────────────────────────────
; hello.asm  — 在 Linux x86-64 上輸出 Hello, World!
;
; 編譯步驟：
;   nasm -f elf64 hello.asm -o hello.o   （組譯）
;   ld hello.o -o hello                   （連結）
;   ./hello                               （執行）
; ─────────────────────────────────────────────

BITS 64                         ; 產生 64-bit 機器碼

; ── 資料區段 ──────────────────────────────────
section .data
    msg     DB  "Hello, World!", 0x0a
    ;           ^─────────────^  ^──
    ;           字串本體（13字元）  換行字元
    
    msg_len EQU $ - msg         ; 字串長度 = 14
    ;           ^   ^
    ;           │   └── msg 的起始位址
    ;           └── 目前位址（msg 結束後）

; ── 程式碼區段 ────────────────────────────────
section .text
    global _start               ; 告訴連結器程式從 _start 開始

_start:
    ; ── 系統呼叫：write(fd, buf, count) ──────
    ; Linux x86-64 系統呼叫約定：
    ;   rax = 系統呼叫編號
    ;   rdi = 第一個參數（fd）
    ;   rsi = 第二個參數（buf 位址）
    ;   rdx = 第三個參數（byte 數）
    
    mov rax, 1                  ; 系統呼叫編號 1 = sys_write
    mov rdi, 1                  ; fd = 1 = stdout（標準輸出）
    mov rsi, msg                ; 第二參數：字串的起始位址
    mov rdx, msg_len            ; 第三參數：要寫入的 byte 數
    syscall                     ; 進入核心，執行系統呼叫

    ; ── 系統呼叫：exit(status) ───────────────
    mov rax, 60                 ; 系統呼叫編號 60 = sys_exit
    xor rdi, rdi                ; 第一參數：結束碼 = 0（成功）
    ;   ↑ xor rdi, rdi 比 mov rdi, 0 短 1 byte，效果相同
    syscall
```

### 2.8.2 指令逐行說明

| 行號 | 程式碼 | 詳細說明 |
|------|--------|---------|
| 1 | `BITS 64` | 指示組譯器產生 64-bit 指令碼，暫存器名稱使用 r 前綴（rax 而非 eax） |
| 2 | `section .data` | 進入已初始化資料區段 |
| 3 | `DB "Hello, World!", 0x0a` | 定義 14 bytes：13 個字元 + 1 個換行（0x0A = LF） |
| 4 | `EQU $ - msg` | `$` 指向 msg 後一個 byte（= msg + 14），相減得 14 |
| 5 | `section .text` | 進入程式碼區段 |
| 6 | `global _start` | 宣告 `_start` 為全域符號，連結器才能找到進入點 |
| 7 | `mov rax, 1` | sys_write 的編號是 1（查閱 Linux syscall table） |
| 8 | `mov rdi, 1` | 第一個參數：fd=1 代表 stdout（0=stdin, 2=stderr） |
| 9 | `mov rsi, msg` | 第二個參數：msg 的位址（不是 msg 的內容，是位址本身） |
| 10 | `mov rdx, msg_len` | 第三個參數：寫入 14 bytes |
| 11 | `syscall` | CPU 切換到核心模式執行系統呼叫，返回後 rax = 寫入的 byte 數 |
| 12 | `mov rax, 60` | sys_exit 的編號是 60 |
| 13 | `xor rdi, rdi` | XOR 自己 = 0，比 `mov rdi, 0` 節省 1 byte 的機器碼 |
| 14 | `syscall` | 呼叫 exit(0)，程式結束 |

### 2.8.3 對應的 C 程式碼

以下 C 程式碼在功能上等同於上述組語：

```c
#include <unistd.h>   // write()
#include <stdlib.h>   // exit()

int main() {
    const char msg[] = "Hello, World!\n";
    write(1, msg, 14);     // 對應組語的 sys_write
    exit(0);               // 對應組語的 sys_exit
    return 0;
}
```

## 2.9 組譯過程：兩遍掃描詳解

### 2.9.1 為什麼需要兩遍？

**問題根源：前向參考（Forward Reference）**

```asm
    JMP  end_label      ; 此時 end_label 還沒定義！
    NOP
    NOP
end_label:              ; 這裡才定義
    RET
```

組譯器在遇到 `JMP end_label` 時，尚不知道 `end_label` 的位址（它在後面才出現），因此無法立刻產生正確的機器碼。解決辦法是**掃描兩遍**。

### 2.9.2 第一遍（Pass 1）：建立符號表

**目標：** 計算每個標號的確切位址，填入符號表。

**運作步驟：**

```
初始化：Location Counter (LC) = 0（或 ORG 指定值）

對每一行原始碼：
    1. 若有標號 → 記錄 (標號, LC 目前值) 到符號表
    2. 判斷指令類型：
       a. 機器指令 → 查表得到指令長度，LC += 長度
       b. DB/DW/DD → 計算資料大小，LC += 大小
       c. RESB/RESD → 計算保留空間，LC += 大小
       d. EQU → 記錄常數值，LC 不變
       e. SECTION → 切換區段，LC 重設或繼續
    3. 繼續處理下一行
```

**範例追蹤：**

```
原始碼                          第一遍處理
─────────────────────────────────────────────────────
                          LC = 0x0000
START:  MOV EAX, 0        → 記錄 START=0x0000，MOV 長 5B → LC=0x0005
        MOV ECX, 5        → LC=0x000A（5B）
LOOP:   ADD EAX, ECX      → 記錄 LOOP=0x000A，ADD 長 2B → LC=0x000C
        DEC ECX           → LC=0x000E（2B）
        JNZ LOOP          → JNZ 長 5B（雖然 LOOP 已知，先記錄）→ LC=0x0013
END:    NOP               → 記錄 END=0x0013，NOP 長 1B → LC=0x0014
        RET               → LC=0x0015（1B）

符號表結果：
  START = 0x0000
  LOOP  = 0x000A
  END   = 0x0013
```

### 2.9.3 第二遍（Pass 2）：產生目的碼

**目標：** 利用符號表，將每條指令翻譯為正確的機器碼。

**運作步驟：**

```
重置 LC = 0

對每一行原始碼：
    1. 查找指令的 opcode
    2. 若運算元包含符號 → 查符號表，取得位址
    3. 計算相對位移或絕對位址
    4. 輸出機器碼 bytes
    5. LC += 指令長度
```

**前向參考的解析：**

```
第一遍時：JNZ LOOP  → LOOP 已知 = 0x000A，記錄但先不填
          JMP END   → END 未知，佔位

第二遍時：JNZ LOOP  → 目標 = 0x000A，計算相對位移，填入正確 byte
          JMP END   → END 已知 = 0x0013，填入正確 byte
```

### 2.9.4 完整兩遍組譯追蹤範例

程式：計算 1+2+3+4+5 的總和

```asm
        ORG  0x0000
START:  MOV  EAX, 0     ; 累加器清零
        MOV  ECX, 5     ; 計數器
LOOP:   ADD  EAX, ECX   ; EAX += ECX
        DEC  ECX        ; ECX--
        JNZ  LOOP       ; ECX≠0 則繼續
        JMP  END        ; 跳到結束（前向參考！）
        NOP             ; 永遠不會執行的指令
END:    HLT             ; 停止
```

**第一遍結果：**

| 位址 | 標號 | 指令 | 長度 | LC 更新後 |
|------|------|------|------|---------|
| 0x0000 | START | MOV EAX, 0 | 5 | 0x0005 |
| 0x0005 | — | MOV ECX, 5 | 5 | 0x000A |
| 0x000A | LOOP | ADD EAX, ECX | 2 | 0x000C |
| 0x000C | — | DEC ECX | 2 | 0x000E |
| 0x000E | — | JNZ LOOP | 5 | 0x0013 |
| 0x0013 | — | JMP END | 5 | 0x0018 |
| 0x0018 | — | NOP | 1 | 0x0019 |
| 0x0019 | END | HLT | 1 | 0x001A |

**符號表：** `START=0x0000, LOOP=0x000A, END=0x0019`

**第二遍輸出（部分）：**

```
位址    機器碼              組語
0x0000  B8 00 00 00 00     MOV EAX, 0
0x0005  B9 05 00 00 00     MOV ECX, 5
0x000A  01 C8              ADD EAX, ECX
0x000C  FF C9              DEC ECX
0x000E  75 FA              JNZ 0x000A  ← 相對位移 = 0x000A - 0x0013 = -9 = 0xF7... 精確計算見下
0x0013  EB 04              JMP 0x0019  ← 相對位移 = 0x0019 - 0x0015 = 4
0x0018  90                 NOP
0x0019  F4                 HLT
```

> **JNZ 位移計算：** 短跳指令使用相對位移，以**下一條指令的位址**為基準。
> `JNZ LOOP`：下一條指令在 0x0013，目標 LOOP=0x000A，位移 = 0x000A - 0x0013 = -9 = 0xF7（8-bit 有號數）

### 2.9.5 符號表的結構

符號表通常以**雜湊表（Hash Table）** 實作，提供 O(1) 的查詢效率：

```
符號表（Symbol Table）
┌───────────────┬──────────┬───────────┬───────────┐
│  符號名稱      │  位址    │  屬性     │  其他資訊 │
├───────────────┼──────────┼───────────┼───────────┤
│  START        │  0x0000  │  GLOBAL   │  .text    │
│  LOOP         │  0x000A  │  LOCAL    │  .text    │
│  END          │  0x0019  │  LOCAL    │  .text    │
│  msg          │  0x0000  │  GLOBAL   │  .data    │
│  msg_len      │  14      │  CONSTANT │  EQU      │
└───────────────┴──────────┴───────────┴───────────┘
```

## 2.10 組語與 C 語言的對應

理解組語的最佳方式之一，是觀察 C 程式碼被編譯成什麼組語：

### 2.10.1 變數宣告

```c
int  a = 10;          // 全域
char msg[] = "Hi";    // 字串
int  buf[100];        // 未初始化陣列
```
```asm
section .data
    a    DD  10
    msg  DB  "Hi", 0

section .bss
    buf  RESD  100
```

### 2.10.2 for 迴圈

```c
int sum = 0;
for (int i = 1; i <= 5; i++) {
    sum += i;
}
```
```asm
    mov eax, 0      ; sum = 0
    mov ecx, 1      ; i = 1
loop:
    add eax, ecx    ; sum += i
    inc ecx         ; i++
    cmp ecx, 5      ; i <= 5?
    jle loop        ; 是則繼續
```

### 2.10.3 if-else

```c
if (eax > 0) {
    ebx = 1;
} else {
    ebx = -1;
}
```
```asm
    cmp eax, 0
    jle else_branch     ; eax <= 0 則跳到 else
    mov ebx, 1          ; if 分支
    jmp end_if
else_branch:
    mov ebx, -1         ; else 分支
end_if:
```

## 2.11 本章小結

本章涵蓋了組合語言的完整基礎，以下是核心要點：

**語法結構：**
- 每行由標號、助記符、運算元、註解四欄位構成
- Intel 語法：`目的 ← 來源`；AT&T 語法：`來源 → 目的`（順序相反）

**程式區段：**
- `.text`：唯讀可執行，存放指令
- `.data`：可讀寫，存放已初始化資料，初始值寫入 ELF 磁碟檔
- `.bss`：可讀寫，存放未初始化資料，磁碟檔只記大小，執行時 OS 清零

**資料定義：**
- DB/DW/DD/DQ：定義各種大小的已初始化資料
- RESB/RESW/RESD/RESQ：在 .bss 保留空間
- EQU：定義不佔記憶體的符號常數
- `$`：目前位址，常搭配 EQU 計算資料長度

**數值表示：**
- x86 使用小端序（Little-Endian）：低位 byte 在低位址
- 有號數以二補數表示，同一 byte 因解讀方式不同意義不同

**兩遍組譯：**
- Pass 1：掃描全文，建立符號表（記錄每個標號的位址）
- Pass 2：翻譯指令，查符號表解析前向參考，輸出機器碼
