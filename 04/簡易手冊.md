## 1. 基本概念

### 系統程式的定義
系統程式（System Programs）是介於硬體與應用程式之間的軟體，負責管理電腦資源並提供開發環境。

| 類型 | 範例 |
|------|------|
| 組譯器 (Assembler) | MASM、NASM、GAS |
| 編譯器 (Compiler) | GCC、Clang |
| 連結器 (Linker) | ld、link.exe |
| 載入器 (Loader) | OS 內建 |
| 巨集處理器 (Macro Processor) | m4、C Preprocessor |
| 作業系統 (OS) | Linux、Windows |

### 程式執行流程

```
原始碼 (.c / .asm)
    ↓ 編譯器 / 組譯器
目的碼 (.o / .obj)
    ↓ 連結器
可執行檔 (.exe / ELF)
    ↓ 載入器
記憶體中執行
```

## 2. 組合語言基礎

### 程式結構（x86 NASM 格式）

```asm
section .data
    msg db "Hello, World!", 0x0a   ; 定義字串資料
    len equ $ - msg                 ; 計算長度

section .bss
    buf resb 64                     ; 保留 64 bytes 未初始化空間

section .text
    global _start

_start:
    mov eax, 4          ; sys_write
    mov ebx, 1          ; stdout
    mov ecx, msg        ; 訊息位址
    mov edx, len        ; 訊息長度
    int 0x80            ; 系統呼叫
```

### 常見虛擬指令（Assembler Directives）

| 指令 | 說明 | 範例 |
|------|------|------|
| `DB` | 定義 Byte | `x DB 10` |
| `DW` | 定義 Word (2 bytes) | `y DW 1000` |
| `DD` | 定義 Double Word (4 bytes) | `z DD 0` |
| `EQU` | 定義常數（不佔記憶體） | `MAX EQU 100` |
| `RESB` | 保留 N bytes | `buf RESB 64` |
| `ORG` | 設定起始位址 | `ORG 100h` |
| `END` | 程式結束 | `END START` |

## 4. 定址模式

| 模式 | 語法（NASM） | 說明 | 範例 |
|------|-------------|------|------|
| 立即值 | `mov eax, 5` | 直接使用常數 | `eax = 5` |
| 暫存器 | `mov eax, ebx` | 暫存器對暫存器 | `eax = ebx` |
| 直接 | `mov eax, [addr]` | 記憶體位址 | `eax = mem[addr]` |
| 間接 | `mov eax, [ebx]` | 暫存器存放位址 | `eax = mem[ebx]` |
| 基底+位移 | `mov eax, [ebx+4]` | 基底加偏移量 | `eax = mem[ebx+4]` |
| 索引 | `mov eax, [ebx+ecx*4]` | 陣列存取 | `eax = arr[ecx]` |

## 5. 指令集架構（ISA）

### 資料傳送指令

```asm
MOV  dst, src     ; dst = src
PUSH src          ; 將值壓入堆疊
POP  dst          ; 從堆疊取出值
XCHG dst, src     ; 交換兩個運算元
LEA  dst, [addr]  ; 載入有效位址
```

### 算術指令

```asm
ADD  dst, src     ; dst = dst + src
SUB  dst, src     ; dst = dst - src
MUL  src          ; AX = AL * src（無號）
IMUL src          ; AX = AL * src（有號）
DIV  src          ; AL = AX / src, AH = 餘數
INC  dst          ; dst++
DEC  dst          ; dst--
NEG  dst          ; dst = -dst
```

### 邏輯與位元指令

```asm
AND  dst, src     ; 位元 AND
OR   dst, src     ; 位元 OR
XOR  dst, src     ; 位元 XOR
NOT  dst          ; 位元 NOT
SHL  dst, n       ; 左移 n 位
SHR  dst, n       ; 右移 n 位（無號）
SAR  dst, n       ; 右移 n 位（有號，保留符號）
ROL  dst, n       ; 左旋轉
ROR  dst, n       ; 右旋轉
```

### 控制流程指令

```asm
JMP  label        ; 無條件跳躍
JE   label        ; ZF=1 時跳（等於）
JNE  label        ; ZF=0 時跳（不等於）
JG   label        ; 大於時跳（有號）
JL   label        ; 小於時跳（有號）
JA   label        ; 大於時跳（無號）
JB   label        ; 小於時跳（無號）
CMP  a, b         ; 比較（設定旗標，不存結果）
CALL label        ; 呼叫副程式
RET               ; 從副程式返回
LOOP label        ; CX-- ; 若 CX≠0 則跳
```

## 6. 連結器與載入器

### 連結過程

```
目的檔 A.o  +  目的檔 B.o  +  程式庫 libc.a
                    ↓  連結器 (ld)
              可執行檔 (ELF / PE)
```

### 重要概念

| 概念 | 說明 |
|------|------|
| **符號表（Symbol Table）** | 記錄函式與變數的名稱與位址 |
| **重定位（Relocation）** | 調整位址，使程式碼在記憶體中正確執行 |
| **靜態連結** | 將程式庫複製進執行檔 |
| **動態連結** | 執行時才載入 `.so` / `.dll` |
| **外部參考（External Reference）** | 本模組用到但定義在其他模組的符號 |
| **入口點（Entry Point）** | 程式開始執行的位址（通常是 `_start` 或 `main`） |

### ELF 檔案結構（Linux）

```
ELF Header
Program Header Table   ← 載入器使用
.text    (程式碼區段)
.rodata  (唯讀資料)
.data    (已初始化資料)
.bss     (未初始化資料)
.symtab  (符號表)
.rel.text (重定位表)
Section Header Table   ← 連結器使用
```

## 7. 巨集處理器

### 巨集定義與使用（NASM 風格）

```asm
; 定義巨集
%macro PRINT 2          ; 巨集名稱、參數個數
    mov eax, 4
    mov ebx, 1
    mov ecx, %1         ; 第一個參數
    mov edx, %2         ; 第二個參數
    int 0x80
%endmacro

; 使用巨集
PRINT msg, len          ; 展開為上述 5 行指令
```

### 巨集 vs 副程式比較

| 比較項目 | 巨集（Macro） | 副程式（Subroutine） |
|----------|--------------|---------------------|
| 執行速度 | 快（無 CALL 開銷） | 較慢 |
| 程式碼大小 | 每次展開一份 | 只有一份 |
| 參數傳遞 | 文字替換 | 暫存器 / 堆疊 |
| 遞迴 | 不支援 | 支援 |

### C 語言巨集（前置處理器）

```c
#define MAX(a, b)   ((a) > (b) ? (a) : (b))
#define SQUARE(x)   ((x) * (x))
#define DEBUG       // 條件編譯開關

#ifdef DEBUG
    printf("Debug mode\n");
#endif
```

> ⚠️ 注意：C 巨集是純文字替換，參數加括號可避免優先順序問題。

## 8. 作業系統介面

### Linux 系統呼叫（x86 32-bit）

| 編號 | 名稱 | EBX | ECX | EDX |
|------|------|-----|-----|-----|
| 1 | `sys_exit` | 結束碼 | — | — |
| 2 | `sys_fork` | — | — | — |
| 3 | `sys_read` | fd | buf 位址 | 長度 |
| 4 | `sys_write` | fd | buf 位址 | 長度 |
| 5 | `sys_open` | 路徑 | flags | mode |
| 6 | `sys_close` | fd | — | — |

```asm
; 系統呼叫範本
mov eax, <呼叫編號>
mov ebx, <第一個參數>
mov ecx, <第二個參數>
mov edx, <第三個參數>
int 0x80
```

### 行程管理概念

| 概念 | 說明 |
|------|------|
| PCB (Process Control Block) | 儲存行程狀態資訊 |
| Context Switch | 切換 CPU 給不同行程 |
| Fork / Exec | 建立 / 替換行程 |
| Signal | 非同步通知機制（如 SIGKILL、SIGTERM） |
| Pipe | 行程間通訊（`|`） |

## 9. 常見工具指令

### GCC 編譯流程

```bash
# 只進行前置處理
gcc -E hello.c -o hello.i

# 編譯為組合語言
gcc -S hello.c -o hello.s

# 組譯為目的碼
gcc -c hello.c -o hello.o

# 完整編譯並連結
gcc hello.c -o hello

# 顯示最佳化的組語
gcc -O2 -S hello.c
```

### 常用分析工具

```bash
# 查看目的檔符號表
nm hello.o

# 反組譯
objdump -d hello.o
objdump -D hello

# 查看 ELF 檔頭
readelf -h hello
readelf -S hello    # 查看所有區段

# 查看動態連結相依
ldd hello

# 追蹤系統呼叫
strace ./hello

# 查看 C 巨集展開
gcc -E -dM - < /dev/null   # 查看所有預定義巨集
```

### NASM 組譯指令

```bash
# 組譯為 32-bit ELF
nasm -f elf32 hello.asm -o hello.o

# 組譯為 64-bit ELF
nasm -f elf64 hello.asm -o hello.o

# 連結（32-bit）
ld -m elf_i386 hello.o -o hello

# 連結（64-bit）
ld hello.o -o hello
```

### 位元運算速記

```
左移 1 位  = × 2       例：0b0001 << 1 = 0b0010  (1 → 2)
右移 1 位  = ÷ 2       例：0b1000 >> 1 = 0b0100  (8 → 4)
x & (x-1) = 清除最低位的 1
x & (-x)  = 取得最低位的 1
x ^ x     = 0（自我 XOR 清零）
~0        = 0xFFFFFFFF（全 1）
```
